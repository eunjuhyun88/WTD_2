## 이번 세션에서 완성한 것

### ✅ RS256 서명 검증 (P2→P0 격상)

**이전 (위험):**
```python
# base64 디코딩만 — 누구나 임의 sub로 토큰 위조 가능
payload_json = base64.urlsafe_b64decode(payload_b64)
```

**이후 (안전):**
```python
public_key = RSAAlgorithm.from_jwk(key_data)  # JWKS에서 공개키 추출
payload = pyjwt.decode(token, public_key, algorithms=["RS256"], audience=..., issuer=...)
```

### ✅ JWT 메트릭 (기존 observability/metrics.py 연동)

```
jwt.cache.hit            → JWKS 캐시 히트 (정상: >99.9%)
jwt.cache.miss           → JWKS fetch 발생 (정상: <0.1%)
jwt.cache.stale_hit      → 구식 캐시 폴백 (비정상 시 Supabase 점검)
jwt.circuit.opened       → 회로 열림 (긴급: >0)
jwt.circuit.recovered    → 자동 복구
jwt.validate.ok          → 성공 검증
jwt.validate.expired     → 만료 토큰
jwt.validate.revoked     → 로그아웃된 토큰
jwt.validate_ms          → 검증 지연시간 (SLO: p99 <500ms)
jwt.jwks_fetch_ms        → JWKS fetch 지연시간
```

### ✅ JSON 구조화 로깅

```json
{"event": "jwt.circuit_transition", "from_state": "CLOSED", "to_state": "OPEN", "failure_count": 5, "ts": "..."}
{"event": "jwt.token_revoked", "sub": "user-uuid", "ts": "..."}
```

### ✅ 토큰 블랙리스트 (Redis)

- `engine/api/auth/token_blacklist.py`
- Redis key: `jwt:revoked:{jti}` (TTL = 남은 토큰 유효시간)
- jti 없으면 sha256(sub:exp) 사용
- Redis 장애 시 soft-fail (가용성 우선)

### ✅ POST /auth/logout

- `engine/api/routes/auth.py`
- Bearer 토큰을 Redis 블랙리스트에 추가
- 이후 동일 토큰 사용 → 403 Token has been revoked

### ✅ k6 부하 테스트 스크립트

- 1000 VU 시나리오 (ramp 30s→100→500→1000, sustain 3min)
