# W-0229 — F-60 Gate

## Goal
카피시그널 Phase 1 진입 조건(verdict 200+ × 5패턴)을 확인하는 엔드포인트와 진행률 UI 제공.

## Owner
engine + app

## Scope
- `engine/api/routes/` `GET /users/{user_id}/f60-status` 신규
- `app/src/routes/dashboard/+page.svelte` F-60 progress bar UI

## Non-Goals
- 카피시그널 실제 기능 (P3, Charter §Frozen 경계)
- Leaderboard / 소셜 기능
- 자동 승격

## Exit Criteria
- `GET /users/{id}/f60-status` → `{verdict_count, pattern_count, gate_open}` 반환
- 조건: `verdict_count >= 200 AND pattern_count >= 5`
- dashboard에 진행률 표시 (예: "37/200 verdicts")
- Engine CI pass, App CI 0 errors

## Facts
1. `engine/ledger/store.py` 또는 Supabase에서 verdict count 집계 가능
2. `engine/patterns/` PatternStateStore에서 active pattern 수 집계 가능
3. 현재 verdict 수 = 0 (초기 상태)

## Assumptions
1. verdict_count는 `valid`+`invalid`+`missed`+`too_late` 합계 (unclear 제외)
2. pattern_count는 사용자가 verdict를 남긴 고유 pattern_slug 수

## Canonical Files
- `engine/api/routes/users.py` 또는 신규 route 파일
- `app/src/routes/dashboard/+page.svelte`
- `app/src/routes/api/users/[id]/f60-status/+server.ts` (신규 프록시)

## 성능 / 보안 설계
- **쿼리**: `SELECT COUNT(*), COUNT(DISTINCT pattern_slug) FROM captures WHERE user_id=? AND verdict NOT IN ('unclear', NULL)` — 인덱스 `(user_id, verdict)` 확인
- **캐시**: 5분 TTL (빈번한 변경 없음) — Supabase Edge Config 또는 in-memory
- **보안**: user_id는 JWT에서 추출, URL 파라미터 user_id와 일치 검증
- **RLS**: captures 테이블 `user_id = auth.uid()` 정책 적용
