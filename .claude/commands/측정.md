---
description: 시간 추이 있는 수치 기록 (지표 추적)
argument-hint: <지표명> <값> [단위]
---

입력: $ARGUMENTS

파싱 절차:
1. 첫 단어 = 지표명 (slug 형태로 정규화: 공백→`-`, 소문자)
2. 두 번째 = 숫자 값
3. 세 번째 (선택) = 단위 (예: ms, %, count)

예시:
- `/측정 search-corpus-precision 0.87` → 지표: search-corpus-precision, 값: 0.87
- `/측정 app-ci-build-seconds 252 s` → 지표: app-ci-build-seconds, 값: 252, 단위: s

대화 맥락에서 자동 추출:
- 측정 환경 (어디서 어떻게 측정했나 — 로컬/CI/프로덕션, 도구)
- 직전 값 (해당 jsonl 파일 마지막 줄에서 읽기)
- 임계치/목표 (있으면 사용자 설정 또는 결정 파일에서)

저장 위치: `memory/metrics/{지표명}.jsonl` (append, 디렉토리 없으면 생성)

레코드 형식:
```json
{"ts":"<KST ISO>","agent":"A###","value":<값>,"unit":"<단위 또는 null>","context":"<요약 1줄>","commit":"<현재 HEAD hex 또는 null>"}
```

저장 후 추이 보고:
- 직전 값 → 현재 값 (변화율 %)
- 5회 이상이면 추세 한 줄 (상승/하락/안정)
- 임계치 있으면 위반 여부 명시

사용자 확인 없이 저장 (단순 append, 의미 기록과 달리 형식이 명확).

저장 후 보고:
```
지표: <name>
값: <prev> → <new> (<변화율>)
임계치: <hit/miss/없음>
파일: memory/metrics/<name>.jsonl
```
