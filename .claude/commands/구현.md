구현 대상: $ARGUMENTS

## 사전 체크 (현재 모델 실행)

```bash
# 1. Spec readiness gate
bash tools/spec-readiness.sh "$ARGUMENTS"
# → score <8이면 즉시 중단, /설계 먼저 실행
```

## Sonnet 구현 위임

spec-readiness score ≥8 확인 후만 아래 Agent 호출:

```
Agent(
  description="구현: $ARGUMENTS",
  model="sonnet",
  prompt="""
아래 사양에 따라 코드를 구현한다.

## 구현 대상
$ARGUMENTS

## Work Item 파일 내용
[spec-readiness.sh 출력에서 Work Item 경로 → 파일 전체 읽기]

## 구현 규칙
1. Work Item의 Canonical Files 목록에 있는 파일만 수정
2. Exit Criteria 항목을 하나씩 체크하며 구현
3. 판단이 필요한 사항은 구현하지 말고 parent에 반환
4. 구현 완료 후 자기검증:
   - Engine 변경: `cd engine && uv run pytest [관련 테스트파일] -x`
   - App 변경: `npm --prefix app run check`
5. 검증 PASS 후 커밋:
   `git add [관련 파일만] && git commit -m "feat(W-XXXX): 한 줄 요약"`

## 반환 형식
구현 완료 시: "DONE — [커밋 SHA] — [검증 결과 요약]"
판단 필요 시: "ESCALATE — [질문 내용]"
검증 실패 시: "FAIL — [에러 내용]"
"""
)
```

## Haiku 결과 처리

- DONE: 커밋 SHA 확인 후 사용자에게 보고
- ESCALATE: 사용자에게 질문 전달
- FAIL: 에러 내용 분석 후 수정 방향 제시 (Haiku 재호출 or 사용자 판단 요청)
