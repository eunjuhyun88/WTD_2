질문: $ARGUMENTS

대화 맥락에서 다음 구조를 자동 추출:

```
질문: $ARGUMENTS

배경:
  (왜 이 질문이 떠올랐나 — 어떤 작업/결정 중이었나)

현재 가설:
  (지금 추측하는 답 — 없으면 "없음"이라고 명시)

검증법:
  - 실험: <돌려볼 수 있는 것>
  또는
  - 측정: <지표 + 임계치>
  또는
  - 문서 확인: <어디 보면 답이 있을 것 같나>
  또는
  - 사용자 확인: <누구한테 물어야 하나>

마감:
  - 즉시 (<24h)
  또는
  - 이번 sprint
  또는
  - 답 나올 때까지 무기한 (블로킹 아님)

블로킹:
  - 이 질문이 답 안 나오면 막히는 것: <work item / PR>
  - 또는 "블로킹 아님"
```

**필수 규칙**:
- 검증법이 "더 생각해보기" 같은 막연한 것이면 거절 — 실행 가능한 수단 필요
- 마감이 "무기한"이고 블로킹도 없으면 "정말 기록할 가치가 있나?" 사용자에게 한 번 더 확인

사용자 [y/수정/취소] 확인 후 저장:
- 경로: `memory/questions/q-{YYYY-MM-DD}-{slug}.md`
- 디렉토리 없으면 `mkdir -p memory/questions`
- frontmatter: `question`, `date`, `agent`, `status` (open), `due`

답이 나오면:
- `/결정`으로 승격해서 결정 파일 생성
- 이 질문 파일의 frontmatter `status`를 `resolved`로, `resolved_by`에 결정 파일 경로 기입

저장 후 보고: 파일 경로.
