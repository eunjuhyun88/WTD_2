description: Work item 완료 마킹 + CURRENT.md/ROADMAP.md 자동 갱신
argument-hint: <W번호 또는 W-슬러그>
---

Work item: $ARGUMENTS

절차:

1. 매칭 파일 찾기:
   - `work/active/W-*$ARGUMENTS*.md` glob
   - 매칭 0개 → "해당 work item 없음" 보고하고 중단
   - 매칭 2개 이상 → 사용자에게 정확한 슬러그 요청

2. 완료 조건 검증:
   - 파일에서 `## Exit Criteria` 또는 `## 완료 조건` 섹션 추출
   - 각 체크박스 항목이 모두 `[x]`인지 확인
   - 미달 항목 있으면 표시 후 사용자에게 [강제 완료/취소] 선택. 강제 완료는 명시적 "강제" 입력 필요

3. 완료 처리:
   - `git mv work/active/W-{slug}.md work/completed/W-{slug}.md`
   - `work/active/CURRENT.md`:
     - 활성 Work Items 표에서 해당 행 제거
     - "✅ 실제로 완료된 것" 표에 행 추가 (근거 = 관련 PR 번호)
   - `spec/ROADMAP.md`에서 해당 줄 제거
   - `mk.log_event(f"W-{slug} 완료", tags=["work","complete"])` 호출

4. 변경 파일 목록 보고:
   ```
   완료: W-<슬러그>
   이동: work/active → work/completed
   갱신:
     - work/active/CURRENT.md
     - spec/ROADMAP.md
   로그: memory/sessions/{date}.jsonl
   ```

5. 추천 다음 액션:
   - 변경 사항 커밋 명령 1줄 제시 (예: `git add ... && git commit -m "..."`)
   - 사용자 승인 없이 commit/push 자동 실행 금지

**필수 규칙**:
- Exit Criteria 미정의 work item은 완료 거절 — 먼저 work item 본문에 완료 조건 명시 요구
- 강제 완료 시에도 미달 항목을 CURRENT.md "근거" 칸에 "(강제 완료, 미달: ...)" 명시
