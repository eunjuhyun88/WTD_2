Work item: $ARGUMENTS

절차:

1. 매칭 파일 찾기:
   - `work/active/W-*$ARGUMENTS*.md` glob
   - 매칭 0개 → "해당 work item 없음" 보고하고 중단
   - 매칭 2개 이상 → 사용자에게 정확한 슬러그 요청

2. 완료 조건 검증:
   - 파일에서 `## Exit Criteria` 또는 `## 완료 조건` 섹션 추출
   - 각 체크박스 항목이 모두 `[x]`인지 확인
   - 미달 항목 있으면 표시 후 사용자에게 [강제 완료/취소] 선택. 강제 완료는 명시적 "강제" 입력 필요

3. **CTO 품질 게이트 검증** (Exit Criteria 통과 후):

   work item의 `## 성능/안정성/보안 결정` 테이블 또는 `## Scope` 항목을 기준으로
   실제 변경된 코드(git diff)에서 다음 4개 축을 grep/read로 직접 확인한다.

   | 축 | 확인 항목 | 통과 기준 |
   |---|---|---|
   | **성능** | N+1 쿼리 여부, SELECT * 여부, limit 누락 여부 | 없음 |
   | **안정성** | fallback/except 처리, 외부 호출 backoff, 멱등 upsert | 명시됨 |
   | **보안** | `requireAuth()` 누락, secret 하드코딩, RLS 미설정 | 없음 |
   | **유지보수** | app↔engine 계층 교차, 통합 테스트 1개 이상 | 교차 없음, 테스트 존재 |

   확인 방법:
   - `git diff main...HEAD -- <Canonical Files>` 로 변경 코드 읽기
   - 각 항목 pass/fail 표시
   - fail 항목은 정확한 파일:라인 표시 + 수정 제안

   **work item에 `## 성능/안정성/보안 결정` 테이블이 없으면**:
   - 표준 4개 축 기준으로 diff를 직접 스캔하여 판단

   미달 항목 있으면 표시 후 [수정 후 재검증 / 강제 완료] 선택.
   강제 완료는 명시적 "강제" 입력 필요.

4. 완료 처리 — `tools/complete_work_item.sh` 호출:

   ```bash
   bash tools/complete_work_item.sh W-{slug}
   # Exit Criteria 미충족 시 강제:
   bash tools/complete_work_item.sh W-{slug} --force
   ```

   스크립트가 자동 처리:
   - `git mv work/active/W-{slug}.md work/completed/W-{slug}.md`
   - `work/active/CURRENT.md` 활성 Work Items 표에서 해당 행 제거
   - `mk.log_event` 호출 (memkraft)

   추가 (스크립트 외):
   - `spec/ROADMAP.md`에서 해당 줄 제거 (수동)
   - 강제 완료 시 CURRENT.md "근거" 칸에 "(강제 완료, 미달: ...)" 명시

5. 변경 파일 목록 보고:
   ```
   완료: W-<슬러그>
   품질 게이트: 성능 ✓ | 안정성 ✓ | 보안 ✓ | 유지보수 ✓  (또는 fail 항목 표시)
   이동: work/active → work/completed
   갱신:
     - work/active/CURRENT.md
     - spec/ROADMAP.md
   로그: memory/sessions/{date}.jsonl
   ```

5. 추천 다음 액션:
   - 변경 사항 커밋 명령 1줄 제시 (예: `git add ... && git commit -m "..."`)
   - 사용자 승인 없이 commit/push 자동 실행 금지

**필수 규칙**:
- Exit Criteria 미정의 work item은 완료 거절 — 먼저 work item 본문에 완료 조건 명시 요구
- 강제 완료 시에도 미달 항목을 CURRENT.md "근거" 칸에 "(강제 완료, 미달: ...)" 명시
