# /컨텍스트 — On-demand Context Pack

작업 설명을 받아 관련 코드·문서·메모리를 8k token 이하로 압축한 Context Pack을 출력한다.

## 사용법

```
/컨텍스트 "V-PV-01 구현"
/컨텍스트 "GateV2 확장"
/컨텍스트 W-0298
/컨텍스트 "차트 그리기 툴"
```

## 실행 순서

### Step 1 — 의도 분류

`$ARGUMENTS`에서 도메인과 work item을 감지한다.

도메인 매칭 규칙:
- `V-*, PV-*, 검증, verify, executor, ledger, pattern, scanner, backtest, stats` → **engine**
- `차트, 터미널, Svelte, UI, app, 프론트` → **app**
- `PR, 머지, worktree, 배포, branch, 에이전트` → **coordination**
- `W-NNNN` 직접 명시 → `work/active/CURRENT.md`에서 도메인 조회

### Step 2 — 컨텍스트 조립

아래 순서로 수집하고 각 레이어를 budget 안으로 유지한다. **총합 ≤ 8,000 tokens**.

1. **Work Item** (~1,000 tokens)
   - `work/active/` 에서 관련 파일 grep → Goal + Scope + Exit Criteria 섹션만 읽기
   - 없으면 `CURRENT.md` active items 테이블 출력

2. **Domain Sub-file** (~500 tokens)
   - engine → `agents/engine.md`
   - app → `agents/app.md`
   - coordination → `agents/coordination.md`

3. **관련 코드** (~3,000 tokens)
   - MCP 사용 가능하면: `mcp__serena__find_symbol` + `mcp__serena__search_for_pattern`
   - fallback: `git grep -n "<keyword>" engine/ app/src/ --include="*.py" --include="*.ts" | head -40`
   - 핵심 함수 정의 + 호출부만 발췌 (파일 전체 읽기 금지)

4. **Domain Doc** (~1,500 tokens)
   - `docs/domains/` 에서 관련 파일 grep → 해당 섹션만

5. **Memory 스니펫** (~500 tokens)
   - `memory/` 에서 task 키워드 grep → 관련 항목 3개 이하

6. **CURRENT.md 요약** (~500 tokens)
   - active items 테이블만 (전체 읽기 금지)

예산 초과 시: 코드 레이어(3)부터 trim. Work Item과 Domain Sub-file은 절대 trim 안 함.

### Step 3 — Pack 출력

아래 포맷으로 출력:

```
━━ Context Pack: <task> ━━━━━━━━━━━━━━━━━━━━━━━━━━━
📋 Work Item: <W-NNNN> — <Goal 1줄>
   Scope: <핵심 파일 목록>
   Exit Criteria: <AC 목록>

🗂  Domain: <engine|app|coordination>
   <agents/<domain>.md 핵심 요약>

🔍 Relevant Code:
   <파일경로:라인> — <함수명/설명>
   ...

📄 Domain Doc:
   <관련 섹션 발췌>

🧠 Memory:
   <관련 메모리 스니펫>

📌 Active Items: <CURRENT.md 테이블>
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
총 추정 토큰: ~N,NNN / 8,000
```

레이어가 비어 있으면 해당 섹션 생략.
