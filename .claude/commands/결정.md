결정 제목: $ARGUMENTS

대화 맥락 + 최근 git log + 관련 PR + 코드를 읽고 다음 구조를 자동으로 채운 초안을 사용자에게 보여주세요:

```
맥락:
  (왜 이 결정이 필요해졌나 — 직접적 원인, 관련 사고/PR)

검토한 옵션:
  1) <옵션 A> — 거절: <구체적 이유>
  2) <옵션 B> — 선택
  3) <옵션 C> — 거절: <구체적 이유>

선택 이유:
  (왜 이 옵션이 다른 옵션들보다 나은가)

영향:
  - 코드: <변경 범위>
  - 운영: <배포/마이그레이션>
  - 타 도메인: <영향 받는 곳>

근거:
  - PR/이슈/커밋: #번호, hex
  - 외부 문서: <있으면 URL/경로>
```

**필수 규칙**:
- 거절된 옵션은 **반드시 거절 이유와 함께** 기록 (다음 에이전트가 같은 옵션 재검토 방지)
- 검토 옵션이 1개뿐이면 "다른 옵션을 검토하지 않은 이유"를 맥락에 명시
- "TBD" "추후 확인" 같은 미정 상태로 저장 금지 — 알 수 없으면 `/물음`으로

사용자 [y/수정/취소] 확인 후 저장:
- 경로: `memory/decisions/dec-{YYYY-MM-DD}-{slug}.md`
- frontmatter: `title`, `date` (KST), `agent` (state/current_agent.txt에서), `tags`
- `mk.add_decision(...)` 호출

저장 후 보고: 파일 경로 + slug.
