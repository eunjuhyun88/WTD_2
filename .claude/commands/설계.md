## PR 분해 필수 원칙

**반드시 적용. 위반 시 설계 반려.**

1. **각 PR은 독립 배포 가능**: 이전 PR 없이도 CI green + 서비스 중단 없음
2. **순서 고정 — shell → data → GTM → 확장**:
   - PR 1: UI 구조만 (mock/placeholder 데이터 허용, 화면이 뜨는 것이 목표)
   - PR 2: 실 API 연결 + migration
   - PR 3: GTM/분석 이벤트 → 이 PR 1주 후 실측으로 AC 수치 확정
   - PR 4+: 실측 데이터 기반 확장 (PR 3 데이터 확인 후 착수)
3. **PR당 파일 수 ≤ 8개**: 초과 시 PR을 쪼갠다
4. **AC 수치 구분**: PR 1-2는 "(est.)" 표기, PR 4+는 "PR 3 실측 후 채움" 표기
5. **검증 포인트 필수**: 각 PR에 "이 PR 머지 후 실제로 알게 되는 것" 1줄 명시
6. **목표치 고정 금지**: AC 수치는 실측 전 추정. 배포 → 측정 → 조정이 루프
