설계 brief: $ARGUMENTS

## 동작 (Sonnet 메인 스레드)

1. **Read 규칙**: `@.claude/commands/_design-router.md`
2. **어휘 매칭**: brief에서 키워드 추출 → role 후보 + lead 결정
3. **5초 확인 1줄**:
   ```
   감지: {role 목록}, lead={lead}. 5초 후 자동 진행 (n 입력 시 중단/수정).
   ```
   - 5초 timeout 또는 응답 대기
4. **Frozen 체크**: `grep -E "Frozen|Non-Goal" spec/CHARTER.md` — 매칭 시 즉시 중단 보고
5. **컨텍스트 수집** (한 번만, 모든 role 공유):
   ```bash
   grep -A5 "$KW" spec/PRIORITIES.md | head -20
   gh issue list --search "$KW" --state all --limit 5
   gh pr list --search "$KW" --state all --limit 5
   git log --all --oneline --grep="$KW" | head -10
   grep -rn "$KW" engine/ app/src/ --include="*.py" --include="*.ts" | head -15
   ls app/supabase/migrations/ | tail -3
   ```

## Dispatch

### N=1 (단일 role)
해당 role 진입점 직접 실행 — `@.claude/commands/설계-{role}.md` 흐름 따라감.

### N≥2 (복수 role)

**Round 1 — 병렬 draft** (각 role Opus 서브에이전트):

```python
Agent(
  description=f"설계 [{role}]: {KW}",
  subagent_type="general-purpose",
  model="opus",
  prompt=f"""
당신은 {role} 전문가.
먼저 Read:
  @.claude/commands/_design-shared.md
  @.claude/commands/_design-{role}.md

## brief
{ARGUMENTS_VERBATIM}

## 컨텍스트 (수집됨)
{PRIORITIES} {ISSUES_PRS} {COMMITS} {CODE_GREP}

## 진행
1. shared + role 규칙 읽기
2. Step 0 캐묻기 4문항 → 답 받음 (안 받으면 진행 X)
3. Step A draft 작성 (role 슬롯, ≤80줄)
4. Hard constraint 1~3개 export
5. parent에 반환: draft + constraints
   (Step B/C 진행 X — parent가 통합)
"""
)
```

**Round 2 — Constraint 충돌 검출** (메인 Sonnet):
- 각 draft에서 `### Hard constraint` 추출
- 자동 충돌 표 생성 (latency 합, cost 합, 시간 의존 reversal)
- 0개 → Round 3 생략, 머지
- ≥1개 → Round 3

**Round 3 — Lead 통합** (lead role Opus):
```python
Agent(
  description=f"설계 통합 [lead={lead}]",
  model="opus",
  prompt=f"""
당신은 {lead} role lead.

## 모든 draft + 충돌 표
{drafts} {conflict_table}

## 진행
1. 충돌 ≤3개에 대해 사용자 결정 질문 (단일 선택지)
2. 답 받으면 통합 설계 ≤200줄 작성:
   - Goal / Scope / Non-Goals
   - 각 role 슬롯 통합
   - PR 분해 (shell→data→GTM→확장)
   - Open Questions
3. Step B 사용자 검토
4. y → Step C: epic Issue + child Issue (role별) 생성, 링크
"""
)
```

## 출력

- 1 epic Issue (lead 기준)
- N child Issue (role별)
- `work/active/W-####-{slug}.md` 저장
- `tools/work_issue_map.sh add W-#### "$ISSUE_NUM" --status design`

`spec/CHARTER.md` Frozen 어휘 매칭 → 즉시 중단, 사용자 보고.
