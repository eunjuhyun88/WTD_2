## Step 1 — 컨텍스트 수집 (현재 모델, 병렬 실행)

다음을 동시에 실행:

```bash
grep -A5 "$ARGUMENTS" spec/PRIORITIES.md | head -20
grep -E "Frozen|Non-Goal|In-Scope" spec/CHARTER.md | head -20
gh issue list --search "$ARGUMENTS" --state all --limit 5
gh pr list --search "$ARGUMENTS" --state all --limit 5
git log --all --oneline --grep="$ARGUMENTS" | head -10
grep -rn "<키워드>" engine/ app/src/ --include="*.py" --include="*.ts" | head -15
ls app/supabase/migrations/ | tail -3
```

**Charter Frozen 항목 발견 시 즉시 중단**, 사용자에게 보고.

수집 결과를 메모리에 저장 → Step 2로 넘김.

---

## Step 2-4 — Opus 서브에이전트 위임

Step 1 컨텍스트 전체를 채워 `Agent(model="opus")` 호출:

```
Agent(
  description="설계문서 + GitHub Issue: $ARGUMENTS",
  model="opus",
  prompt="""
CTO + AI Researcher 2-perspective 설계 에이전트입니다.
아래 컨텍스트를 기반으로 설계문서를 작성하고 사용자 검토 후 Issue를 생성합니다.

## 컨텍스트 (Step 1 수집)

{PRIORITIES 결과}
{CHARTER 결과}
{중복 이슈/PR}
{관련 커밋}
{코드 실측}

## 설계 대상

$ARGUMENTS

## 제약

- Frozen: copy_trading, leaderboard, AI 차트 분석, 자동매매, 신규 메모리 stack
- TBD 금지 — 모르면 Open Questions
- Exit Criteria 수치 없으면 반려
- 거절 옵션은 거절 이유 기록

## Step A — 설계 초안 작성 (아직 파일 저장 안 함)

형식:

# W-#### — {Title}

> Wave: {N} | Priority: {P0/P1/P2} | Effort: {S/M/L}
> Charter: {In-Scope L#}
> Status: 🟡 Design Draft
> Created: {YYYY-MM-DD}

## Goal
사용자 입장에서 무엇이 가능해지는가 (1줄).

## Scope
- 포함: ...
- 파일: engine/... app/...
- API: POST /...

## Non-Goals
- (이유 포함)

## CTO 관점
### Risk Matrix
| 리스크 | 확률 | 영향 | 완화 |
|---|---|---|---|

### Dependencies / Rollback / Files Touched (실측 기반)

## AI Researcher 관점
### Data Impact / Statistical Validation / Failure Modes

## Decisions
- [D-####] 선택 옵션 (거절 옵션 + 이유)

## Open Questions
- [ ] [Q-####] ?

## Implementation Plan
1. ...
2. ...
3. 테스트

## Exit Criteria
- [ ] AC1: 수치 포함 필수
- [ ] CI green
- [ ] PR merged + CURRENT.md SHA 업데이트

## Step B — 사용자 검토

초안을 사용자에게 보여주고 응답 대기:
- y → 저장 + Step C
- 수정 → 섹션 지정 후 재작성
- 취소 → 종료 (파일 저장 안 함)

## Step C — 저장 + Issue 생성 (Atomic, y 응답 시에만)

Issue 생성 실패 시 파일 저장 없이 중단:

```bash
ISSUE_URL=$(gh issue create \
  --title "[Wave-{N}] W-#### — {Title}" \
  --body "## Goal\n{1줄}\n\n## Exit Criteria\n{AC 목록}\n\nSee work/active/W-####-{slug}.md" \
  --label "{wave-N},{priority}")
ISSUE_NUM=$(echo "$ISSUE_URL" | grep -oE '[0-9]+$')
```

파일 저장: frontmatter에 `> Issue: #N` 추가.
map 등록: `tools/work_issue_map.sh add W-#### "$ISSUE_NUM" --status design`
보고: 파일 경로 + Issue URL
"""
)
```
