반복 작업 대상: $ARGUMENTS

## 적합한 작업 유형

- 파일명 일괄 rename / slug 변환
- 상수 초안 목록 생성 (판단 없이 패턴 추출)
- Test fixture 파일 생성 (기존 패턴 복사·변형)
- 단순 패턴 치환 (`sed`/`grep` 수준)
- 파일 목록 수집 + 분류

## 부적합 (Haiku 금지 — /구현 사용)

- 코드 작성·리팩토링
- 테스트 논리 작성
- 아키텍처 판단
- 에러 디버깅

## 구현 지시 (Haiku가 직접 실행)

1. 작업 목록을 먼저 echo로 나열 (dry-run 확인)
2. 사용자 확인 없이 진행 (반복 작업은 즉시 실행)
3. 완료 후: 처리 파일 수 + 미처리 이유 반환
4. 추가 판단 필요 시: parent에 반환 (에스컬레이션 금지)
5. 추가 서브에이전트 생성 금지

## 완료 형식

```
DONE — N개 파일 처리 완료
처리: [파일목록 요약]
미처리: [이유] (있을 경우)
```
