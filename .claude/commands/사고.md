사고 제목: $ARGUMENTS

대화 맥락 + 에러 로그 + 수정 commit을 읽고 다음 구조를 자동으로 채운 초안을 사용자에게 보여주세요:

```
증상:
  (사용자 또는 시스템이 본 것 — 구체적 에러 메시지/실패 동작)

근본 원인:
  (메커니즘 — "환경 탓" 아니라 "왜 이 코드가 이 입력에 이렇게 반응했나")

수정:
  - 변경 파일: <path:line>
  - 커밋: <hex>
  - 핵심 변경: <한 줄>

재발 방지:
  - 테스트 추가: <경로 또는 X>
  - 계약 갱신: <design/current/contracts/*.md 갱신 여부>
  - lint/CI 룰: <있으면>
  - 운영 가드: <있으면>

타임라인:
  - 발생: <시각>
  - 감지: <시각, 누가/어떻게>
  - 수정: <시각>
```

**필수 규칙**:
- 근본 원인은 "왜"를 5번 정도 파야 함 — 표면 원인("X가 null이었음") 금지, 메커니즘("Y 호출 순서가 보장 안 됨")까지
- 재발 방지는 **실행 가능한 수단**만 (테스트/CI/계약). "주의하기" 같은 인적 약속 금지

사용자 [y/수정/취소] 확인 후 저장:
- 경로: `memory/incidents/inc-{YYYY-MM-DD}-{slug}.md`
- frontmatter: `title`, `date` (KST), `agent`, `tags`, `severity` (P0/P1/P2)
- `mk.add_incident(...)` 호출

저장 후 보고: 파일 경로.
