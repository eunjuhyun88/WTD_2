## 순서 (이 순서 엄수, 건너뛰기 금지)

### 1. 인계 초안

```bash
git log --oneline --since="8 hours ago"
git diff --stat origin/main..HEAD
```

아래 5개 항목으로 초안 작성 → 사용자 [y/수정] 확인:

```
한 일:      PR #N / commit SHA — 한 줄 요약
안 한 일:   의도적 Out-of-Scope
막힌 곳:    블로커 또는 미확인 사항
즉시 다음:  다음 에이전트 첫 5분 안에 할 명령 1줄
전제 가정:  main SHA, 환경 변수, 의존 PR 상태
```

```bash
grep -oE "(F|H|V|A|D|N|L|S|C)-[0-9]+" work/active/W-*.md 2>/dev/null | sort -u
grep -n "<ID>" spec/PRIORITIES.md
```

편집 규칙:
- P0/P1 해당 행 → `~~취소선~~`

### 5. 다음 work item 설계 (필수 — 없으면 종료 거부)

`spec/PRIORITIES.md` P0/P1 중 `work/active/W-XXXX-*.md` 없는 항목 → 신규 생성.

**작성 전 코드 실측 필수:**
```bash
grep -rn "<기능 키워드>" engine/ app/src/ --include="*.py" --include="*.ts" | head -10
ls app/supabase/migrations/ | tail -3
```

**필수 섹션** (수치 없는 Exit Criteria = 반려):
```
## Goal           — 사용자 가치 1문장 (Jin 페르소나)
## Owner          — engine / app / engine+app
## Scope          — 파일 목록 + 변경 이유 (실측 기반)
## Non-Goals      — 명시적 제외
## Exit Criteria  — 수치 기준 필수 (예: p95 < 200ms, accuracy ≥ 0.55)
## AI Researcher 리스크  — 훈련 데이터 영향 + 통계 유효성
## CTO 설계 결정  — 성능/안정성/보안/유지보수성
## Facts          — grep 실측 결과
## Canonical Files — 건드릴 파일 전체
```

### 7. 장기 메모리 보존 (필수 — 없으면 종료 거부)

**파일 생성** (`~/.claude/projects/-Users-ej-Projects-wtd-v2/memory/`):

```
project_session_<agentid>_<slug>_<YYYYMMDD>.md
```

필수 섹션:
```
## 산출물     — PR/SHA/CI 결과
## 처리 내역  — 1줄 요약 목록
## 사용자 결정 — lock-in 결정 (영구 가치)
## lesson    — 재발 방지 노트
## 다음 P0   — 후속 에이전트 픽업 후보
## 검증 SHA  — baseline → final main SHA
```

**MEMORY.md 최상단 추가** (`## Current State (YYYY-MM-DD — Latest)` 안에):
```
- [파일명.md](파일명.md) — **한줄 요약**. 핵심 결정 + lesson.
```

⚠️ MEMORY.md 200줄 제한 — entry는 3줄 이내.
