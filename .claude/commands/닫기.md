세션 종료 절차 (인계 작성 강제):

1. 이번 세션의 활동을 자동 추출:
   - `git log` 이번 세션 동안의 커밋
   - 생성/머지된 PR 목록
   - 변경된 파일 (`git diff --stat`)
   - 대화 맥락에서 "한 일 / 안 한 일 / 막힌 곳" 식별

2. `/인계` 슬래시 명령어를 호출한 것처럼 다음 구조로 초안 작성:
   - 한 일 (구체적 PR 번호/커밋 해시 포함)
   - 안 한 일 (의도적으로 스코프 외였던 것)
   - 막힌 곳 (블로커, 알 수 없었던 것)
   - 즉시 다음 단계 (다음 에이전트가 첫 5분 안에 할 일)
   - 전제 가정 (main SHA, 의존 PR 상태, 환경 — 깨지면 무효)

3. 사용자에게 초안 보여주고 [y/수정/취소] 확인. 수정 요청 시 한 줄씩 반영.

4. 확인되면:
   - `bash tools/end.sh "<shipped 한 줄>" "<next 한 줄>" "<lesson 한 줄 또는 빈 문자열>"` 실행
   - 추가로 `memory/sessions/agents/<Agent ID>.jsonl`에 위 5개 필드 전체를 JSON으로 append
   - `spec/CONTRACTS.md`에서 자기 lock 행 제거 (end.sh가 처리하지만 검증)

5. 종료 보고: 저장된 파일 경로 + 세션 요약 1줄.

인계 작성 없이는 종료 거부.
