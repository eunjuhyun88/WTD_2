세션 종료 절차 (인계 + 정리 + 다음 설계 강제):

1. 이번 세션의 활동을 자동 추출:
   - `git log` 이번 세션 동안의 커밋
   - 생성/머지된 PR 목록
   - 변경된 파일 (`git diff --stat`)
   - 대화 맥락에서 "한 일 / 안 한 일 / 막힌 곳" 식별

2. `/인계` 슬래시 명령어를 호출한 것처럼 다음 구조로 초안 작성:
   - 한 일 (구체적 PR 번호/커밋 해시 포함)
   - 안 한 일 (의도적으로 스코프 외였던 것)
   - 막힌 곳 (블로커, 알 수 없었던 것)
   - 즉시 다음 단계 (다음 에이전트가 첫 5분 안에 할 일)
   - 전제 가정 (main SHA, 의존 PR 상태, 환경 — 깨지면 무효)

3. 사용자에게 초안 보여주고 [y/수정/취소] 확인. 수정 요청 시 한 줄씩 반영.

4. **머지된 work item 자동 정리** (AGENTS.md:192 enforcement):

   이번 세션 또는 origin/main에서 머지된 PR을 식별하고 매핑된 W-XXXX work item을
   `work/completed/`로 자동 이동한다.

   절차:
   - `git log --merges --since="<session_start>" --pretty="%H %s"` 또는
     `git log origin/main --oneline -20` 검토
   - 각 머지 commit/PR에서 정규식 `W-\d{4}` 추출
   - 매칭된 `work/active/W-XXXX-*.md` 표시
   - 사용자 확인 후 각각 `bash tools/complete_work_item.sh W-XXXX` 호출
   - Exit Criteria 미충족이면 `--force` 옵션 안내 (강제 완료)
   - 결과: `git mv` + `CURRENT.md` 활성표 행 제거 + memkraft log

   대상이 없으면 스킵.

5. **Sweep + Drift Check** (자동 정리 + 경고):

   `tools/end.sh`가 실행 중 자동으로 다음을 호출한다 (수동 호출 불필요):
   - `tools/sweep_session_artifacts.sh` — D-1 세션 아티팩트 archive, D-2 docs/live 중복 삭제
   - `tools/check_drift.sh` — E-1 main SHA, E-2 W-#### 충돌, E-3 활성표 모순, E-4 origin/main 차이

   결과 보고를 사용자에게 전달.

6. **CTO + AI Researcher 관점으로 다음 work item 설계** (종료 전 필수):

   `tools/check_drift.sh` 의 E-2 결과(다음 사용 가능 W-#### 번호)를 반영해서 충돌 없는 번호로 생성.

   `spec/PRIORITIES.md`의 P0/P1/P2 중 `work/active/W-XXXX-*.md` 파일이 없는 항목을 찾아
   각각 `work/active/W-XXXX-<slug>.md`를 생성한다.

   파일을 생성할 때 다음 CTO 설계 원칙을 적용한다:

   ### 성능 (100명+ 동시 사용자 기준)
   - DB 쿼리: N+1 금지, 인덱스 명시, bulk upsert 선호
   - Supabase: `select()` 컬럼 명시 (SELECT * 금지), limit 항상 지정
   - 캐시 전략: hot path에 Redis/Edge Config 여부 판단
   - 비동기: blocking I/O는 `asyncio.to_thread()` 래핑

   ### 안정성
   - 폴백: Supabase 장애 시 file fallback 또는 graceful degradation 명시
   - 멱등성: upsert는 conflict key 명시
   - 재시도: 외부 API 호출에 exponential backoff 명시
   - 헬스체크: 새 경로마다 `/health` 또는 observability endpoint 포함

   ### 보안
   - JWT: 모든 app API route에 `requireAuth()` 적용 여부 명시
   - RLS: Supabase 테이블마다 Row Level Security 정책 명시
   - 민감값: secret은 env var, 코드 리터럴 금지
   - 입력 검증: 외부 입력은 zod(app) 또는 pydantic(engine) 스키마 통과 후 사용

   ### 유지보수성
   - 계층 준수: `app/`은 surface, `engine/`은 business logic — 교차 금지
   - 계약: app ↔ engine 경계에 OpenAPI 또는 typed contract 정의
   - 테스트: 핵심 경로마다 통합 테스트 1개 이상 명시
   - 롤백: migration은 down script 또는 non-destructive 전략 명시

   ### Charter 정합성 확인
   - `spec/CHARTER.md` In-Scope(L3–L7 갭) 안에 있는지 확인
   - Non-Goal 키워드(copy_trading, chart_polish, memkraft 신규, multi-agent 신규) 진입 금지
   - 위반 시 scope를 In-Scope로 재조정 후 작성

   ### work item 필수 섹션
   ```
   ## Goal          — 1문장, 사용자 가치 중심
   ## Owner         — engine / app / engine+app
   ## Scope         — 변경 파일 목록 + 각 변경 이유
   ## Non-Goals     — 이번 work item에서 명시적으로 제외
   ## Exit Criteria — 완료 판단 기준 (테스트 pass, row count, UI 노출 등)
   ## Facts         — 현재 코드 상태 (grep 결과, migration 존재 여부 등)
   ## Assumptions   — 환경 가정 (env var 설정, migration 실행 여부 등)
   ## Canonical Files — 건드릴 파일 전체 목록
   ```

   이미 파일이 있으면 스킵 (덮어쓰기 금지).
   생성한 파일 목록을 사용자에게 표시.
   **다음 work item 파일 없이는 종료 거부.**

7. 확인되면:
   - `bash tools/end.sh "<shipped 한 줄>" "<next 한 줄>" "<lesson 한 줄 또는 빈 문자열>"` 실행
   - end.sh가 자동으로 sweep_session_artifacts.sh + check_drift.sh 호출 (Step 5)
   - 추가로 `memory/sessions/agents/<Agent ID>.jsonl`에 위 5개 필드 전체를 JSON으로 append
   - `spec/CONTRACTS.md`에서 자기 lock 행 제거 (end.sh가 처리하지만 검증)

8. 종료 보고:
   - 저장된 파일 경로 + 세션 요약 1줄
   - 정리된 work item 카운트 (Step 4 결과)
   - sweep/drift 경고 카운트 (Step 5 결과)
