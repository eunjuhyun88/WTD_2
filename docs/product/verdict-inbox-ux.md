# Verdict Inbox — Trader UX Design

> **작성 기준**: CTO + Trader + Designer 관점. 트레이더가 실제로 판단하는 방식으로 설계.
> **적용 서피스**: Cogochi 좌측 VERDICTS 탭 + Dashboard 주간 리뷰

---

## 핵심 통찰

트레이더는 "알림 inbox"를 처리하지 않는다.
트레이더는 **"내 thesis가 맞았는가"를 확인**한다.

```
나쁜 UX:  "Signal Alert — BTC 4h — pending"
좋은 UX:  "tradoor_v2 / OI급등 후 번지대
           → ACCUMULATION 구간 진입 확인됨
           → 이후 4h +4.2%  ← 맞았나?"
```

### 카드 정보 계층

```
1순위 (한눈에):  패턴명 + 결과 방향 + % 변동
2순위 (이해):   thesis (저장 당시 메모) + 감지된 페이즈
3순위 (판단):   Agree / Disagree / Missed 3개 버튼만
```

### 판정 버튼 규칙

| 버튼 | 의미 | 색상 |
|------|------|------|
| ✓ 유효 | thesis가 맞았고, 패턴이 작동함 | 초록 |
| ✗ 무효 | thesis가 틀렸거나, 노이즈였음 | 빨강 |
| ~ 놓침 | 패턴은 맞았지만 진입 못함 | 회색 |

- 버튼 클릭 → 즉시 판정 저장 (confirm 없음)
- 판정 후 → 카드가 아래 "완료" 섹션으로 슬라이드
- 실수 판정 → 완료 섹션에서 3초 이내 undo 가능

### 중앙 차트 연동

- VERDICTS 탭에서 카드 클릭 → 중앙 차트가 해당 심볼/구간으로 이동
- 차트 보면서 판정 버튼 클릭 가능 (탭 유지)
- 판정 후 → 다음 대기 카드로 자동 포커스

```
클릭 → 해당 패턴 판정 내역 전체 보기
```

## 정보 계층 원칙

### 트레이더 시선 순서

```
1. 결과가 좋았나? (색상 + %)
   → 초록/빨강 + 숫자로 즉시 파악

2. 왜 저장했었나? (thesis)
   → 저장 당시 메모가 그대로 보임

3. 실제로 뭐가 일어났나? (감지 페이즈)
   → "ACCUMULATION 감지됨" 한 줄

4. 내 판단은? (버튼 3개)
   → 더 이상 읽을 것 없음
```

### 절대 보여주지 말 것

- raw capture ID
- JSON 필드명
- "signal alert ID #1234"
- 날짜/시간 (상대 시간만: "3시간 전")
- confidence 숫자 (badge만: Verified / — )

## 상태별 UX

| 상태 | 표시 |
|------|------|
| 대기 판정 | 카드 왼쪽에 ● 오렌지 도트 |
| 유효 판정 완료 | ✓ 초록 + 수치 + 접힘 |
| 무효 판정 완료 | ✗ 빨강 + 수치 + 접힘 |
| 놓침 | ~ 회색 + 접힘 |
| 자동 판정 (outcome resolver) | 🤖 아이콘 + "자동" 라벨 |

**수동 판정 vs 자동 판정 구분 필수**:
- 수동: 유저 아이콘
- 자동: 🤖 아이콘
- 두 판정이 충돌할 경우 → 수동 판정 우선 + 차이 표시

## 빈 상태 (Empty States)

| 섹션 | 빈 상태 copy |
|------|-------------|
| 대기 판정 없음 | "모두 판정 완료. 다음 패턴 알림을 기다리는 중." |
| 완료 없음 | "저장한 셋업이 트리거되면 여기 나타납니다." |
| 주간 성과 없음 | "이번 주 판정 데이터가 없습니다. Terminal에서 셋업을 저장하세요." |

---

## 모바일 최적화

트레이더는 **판정은 모바일**, **분석은 데스크탑**으로 한다.

모바일 VERDICTS:
- 카드 스와이프: 오른쪽 → 유효, 왼쪽 → 무효
- 텍스트 최소화, 버튼 최대화
- 차트 썸네일 (탭하면 확대)

## 구현 우선순위

| 우선순위 | 항목 |
|---------|------|
| P0 | 판정 3버튼 (유효/무효/놓침) + 즉시 저장 |
| P0 | thesis 텍스트 + 결과 % 표시 |
| P1 | 중앙 차트 연동 (카드 클릭 → 차트 이동) |
| P1 | 자동/수동 판정 구분 표시 |
| P2 | 패턴별 성과 카드 (Dashboard 주간 리뷰) |
| P2 | Refinement 추천 (Lab 연동) |
| P3 | 모바일 스와이프 판정 |
| P3 | 3초 undo |
