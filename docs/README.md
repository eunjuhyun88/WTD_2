# Docs Router

This docs tree is layered for low-context operation.

## Read Order

1. `docs/product/*.md` for current product truth
2. `docs/domains/*.md` for area ownership and maps
3. `docs/decisions/*.md` for decision rationale
4. `docs/runbooks/*.md` for execution procedures

## Non-Canonical / Historical

- `docs/archive/` is excluded by default for coding tasks.
- Legacy docs under `app/docs/` are reference material unless explicitly promoted here.
