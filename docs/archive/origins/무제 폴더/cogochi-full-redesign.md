## 0. 제품 정체성 (모든 설계의 출발점)

### 한 문장 정의
> **COGOCHI는 게임을 하면서 나만의 트레이딩 AI 모델을 훈련시키는 다마고치다.**

### 작동 원리
```
플레이어가 게임을 한다 (시그널 판단, Arena 배틀, 패턴 분석)
        ↓
그 행동이 훈련 데이터가 된다 (판단 기록, 승패, 컨텍스트)
        ↓
에이전트(소형 모델)가 파인튜닝된다 (ORPO/DPO/규칙 기반 학습)
        ↓
에이전트의 분석 능력이 실제로 좋아진다
        ↓
더 좋은 분석 → 더 좋은 판단 → 더 좋은 훈련 데이터 → 선순환
```

### 3줄 가치 명제
1. **게임이 곧 훈련이다** — 시간 낭비가 아님. 놀수록 AI가 좋아짐
2. **내 모델은 나만의 것이다** — 내 판단 스타일, 내 리스크 성향이 반영된 유일한 모델
3. **성장이 진짜다** — 가짜 레벨업이 아니라 실제 모델 성능이 향상됨

### 제품 레이어

```
Layer 1: 다마고치 (감정)
  에이전트 캐릭터, 반응, 성장 시각화, 돌봄
  → 왜 계속 하는가? "얘가 보고 싶어서"

Layer 2: 트레이딩 게임 (재미)
  시그널 판단, Arena 배틀, 실시간 시장 데이터
  → 왜 재미있는가? "맞추면 짜릿하니까"

Layer 3: AI 모델 훈련 (가치)
  ORPO 파인튜닝, 패턴 메모리, 레짐 적응
  → 왜 가치있는가? "내 AI가 실제로 똑똑해지니까"

Layer 4: 모델 소유권 (자산)
  개인 소형 모델 소유, 성능 증명, 거래 가능
  → 왜 떠나지 않는가? "이건 내가 키운 자산이니까"
```

## 1. 페르소나 (JTBD 기반) [추정]

### 페르소나 1: "진(Jin)" — 차트 보는 개인 트레이더

**기본 정보**
- 28세, 크립토 2~3년차, 바이낸스/바이비트 사용
- 하루 1~3시간 차트 분석. 레버리지 트레이딩 경험 있음
- 트위터/텔레그램에서 시그널 팔로우 중

**Primary JTBD**
"내 트레이딩 판단의 정확도를 높이고 싶다.
근데 AI 봇은 블랙박스라 믿기 어렵고,
내가 직접 쓰면서 내 스타일에 맞는 AI를 갖고 싶다."

**Top 3 Pains**
1. **AI 봇이 블랙박스** — 3Commas/Pionex 써봤지만 왜 그렇게 판단하는지 모름. 돈 잃으면 분노만 남음 (Critical)
2. **시그널 과부하** — 텔레그램 10개 채널 팔로우하는데 서로 반대 말함. 뭘 믿어야 하는지 모름 (Significant)
3. **복기를 안 함** — 매매일지 쓰려고 하는데 귀찮아서 안 함. 같은 실수 반복 (Significant)

**Top 3 Gains**
1. 내 판단 패턴을 객관적으로 보고 싶다 (언제 맞고 언제 틀리는지)
2. AI가 내 약점을 보완해주면 좋겠다 (내가 못 보는 걸 잡아주는)
3. 점점 좋아지는 게 눈에 보이면 좋겠다 (승률 60% → 65% → 70%)

**Unexpected Insight** [추정]
진은 "AI가 대신 트레이딩 해주기"를 원하지 않음. **"내가 더 잘하게 도와주기"**를 원함. 자율성(Autonomy)이 핵심. 봇에 돈 맡기는 건 불안하지만, 내 판단에 AI가 힘을 실어주는 건 환영.

**COGOCHI 적합성**
- Pain 1 해결: 내 데이터로 훈련된 모델이니 블랙박스가 아님
- Pain 3 해결: 게임 플레이 자체가 매매일지 + 복기
- 마찰: Terminal의 초기 복잡도 (Bloomberg급 UI)

### 페르소나 2: "미나(Mina)" — AI에 호기심 있는 크립토 초보

**기본 정보**
- 24세, 크립토 6개월차, 업비트/코인베이스 현물만
- ChatGPT 헤비 유저. "AI로 뭔가 할 수 있지 않을까?" 관심
- 틱톡/유튜브에서 크립토 정보 습득

**Primary JTBD**
"크립토 트레이딩을 배우고 싶은데 너무 어렵다.
AI가 도와주면서 재밌게 배울 수 있으면 좋겠다.
근데 진짜 돈 걸기는 무섭다."

**Top 3 Pains**
1. **학습 곡선이 너무 가파름** — RSI, MACD, OI... 용어만 들어도 포기 (Critical)
2. **실전 연습할 곳이 없음** — 데모 트레이딩은 긴장감 0. 진짜 돈 넣기엔 무서움 (Significant)
3. **혼자 하면 지루함** — 차트 보는 건 재미없음. 3일 만에 포기 (Significant)

**Top 3 Gains**
1. 게임처럼 재밌게 트레이딩 감각을 익히고 싶다
2. 내가 기른 AI가 나중에 진짜 쓸모있으면 좋겠다
3. 커뮤니티에서 다른 사람들이랑 비교하며 성장하고 싶다

**Unexpected Insight** [추정]
미나의 진짜 동기는 트레이딩이 아니라 **"나만의 AI를 갖는 것"**.
ChatGPT 세대에서 "내가 직접 훈련시킨 AI"는 강력한 소유 욕구를 자극함.
다마고치 감성이 가장 잘 먹히는 페르소나.

**COGOCHI 적합성**
- Pain 1 해결: 게임이 학습을 대체 (차트 이론 몰라도 시그널 판단으로 시작)
- Pain 2 해결: Arena = 실전 긴장감 + 실제 BTC 데이터인데 돈은 안 걸림
- Pain 3 해결: 다마고치 + 커뮤니티
- 마찰: Terminal이 여전히 어려움 → 게임 UI로 감싸야 함

### 페르소나 3: "덱스(Dex)" — 알파 헌터 / 크립토 파워유저

**기본 정보**
- 32세, 크립토 5년+, DeFi 네이티브
- 온체인 분석, 고래 추적, MEV 봇 돌려본 경험
- 뭐든 엣지가 되면 쓰고, 안 되면 바로 버림

**Primary JTBD**
"남들이 못 보는 알파를 찾고 싶다.
AI가 내 분석 위에 추가 레이어를 얹어주면
진짜 엣지가 될 수 있다."

**Top 3 Pains**
1. **기존 AI 봇은 다 똑같음** — 모두 같은 모델, 같은 데이터. 엣지 없음 (Critical)
2. **신호 대 잡음 비율** — 정보는 넘쳐나는데 진짜 알파는 1%도 안 됨 (Significant)
3. **커스텀 모델 구축 비용** — 직접 파인튜닝하려면 GPU 비용 + 데이터 + 시간 (Critical)

**Top 3 Gains**
1. 나만의 엣지가 있는 커스텀 AI 모델
2. 모델의 성능이 증명 가능 (verifiable track record)
3. 좋은 모델을 자산으로 소유/거래 가능

**Unexpected Insight** [추정]
덱스는 다마고치 UI를 **처음엔 무시**함. "유치하다"고 생각.
하지만 자기 모델의 승률이 올라가기 시작하면 → **"이 오리가 진짜 똑똑해지네"** → 감정 이입.
덱스를 잡으려면 **데이터로 증명**이 먼저, 감정은 나중에 따라옴.

**COGOCHI 적합성**
- Pain 1 해결: 각자의 데이터로 파인튜닝 → 모든 모델이 다름 = 유니크 엣지
- Pain 3 해결: 게임 하면서 자동으로 훈련 데이터 축적 → GPU 비용 없이 모델 개선
- 마찰: "이게 진짜 작동해?" 증명 필요. Oracle 리더보드 + 성능 메트릭 투명성 필수

```

### Direct 경쟁사

**1. 3Commas / Pionex** — AI 트레이딩 봇
- 3Commas: $79/월, 사전 설정 전략, 백테스트 제공
- Pionex: 무료 봇 16개, 0.05% 수수료
- 강점: 검증된 플랫폼, 거래소 직접 연동, 실제 자동매매
- 약점: **블랙박스** (왜 그렇게 판단하는지 모름), 모든 유저 같은 모델, 학습/성장감 없음
- COGOCHI 차별점: **내 데이터로 훈련, 과정이 투명, 성장이 보임**

**2. AIXBT** — AI 에이전트 토큰 (MC ~$24M, 87% 하락 from ATH)
- KOL 400+ 트래킹, 자율 시장 인텔리전스
- 강점: 24/7 자율 운영, 실시간 인사이트
- 약점: **하나의 범용 모델** (개인화 없음), 토큰 의존 (가격 하락 시 관심 ↓), 유저가 읽기만 함 (참여 없음)
- COGOCHI 차별점: **각자의 모델, 참여형 훈련, 토큰 가격 무관한 제품 가치**

### Indirect 경쟁사

**3. Aavegotchi / PlayDoge** — 크립토 다마고치
- Aavegotchi: DeFi + 다마고치, aToken 스테이킹으로 가치 백업
- PlayDoge: $PLAY 토큰 P2E, 프리세일 $4.3M
- 강점: 다마고치 감성, 크립토 네이티브, 커뮤니티 강함
- 약점: **토큰 인센티브 제거하면 할 이유 없음**, AI 없음, 게임이 얕음
- COGOCHI 차별점: **게임의 결과물이 실제 AI 모델** (토큰 없어도 가치)

**4. TradingView** — 차트 분석 도구
- MAU 90M+, 프리미엄 $12.95~$59.95/월
- 강점: 차트 분석의 표준, 소셜 기능, 방대한 유저 베이스
- 약점: **AI 없음**, 분석 도구일 뿐 판단/학습 안 해줌, 게임성 0
- COGOCHI 차별점: **분석 + 판단 + 학습이 하나의 루프**

### Substitute (현재 유저가 실제로 하는 것)

**5. 텔레그램 시그널 채널 + 수동 트레이딩**

- 약점: **KOL 의존** (KOL 틀리면 끝), 학습 안 됨, 복기 안 함
- COGOCHI 차별점: **내가 직접 판단 + AI가 복기 = 실력 성장**

## 4. GTM 전략

### ICP 우선순위

```
1순위: 진(Jin) — 차트 보는 개인 트레이더
  → Pain이 가장 극심 (블랙박스, 복기 안 함)
  → 돈 쓸 의향 있음 (이미 3Commas $79 내본 사람들)
  → 만족 시 → 덱스(Dex)로 자연 확장

2순위: 미나(Mina) — AI 호기심 크립토 초보
  → 수는 가장 많지만 전환율 낮음
  → 다마고치 바이럴의 주역 (SNS 공유)
  → 무료 티어로 유입 → 프리미엄 전환

3순위: 덱스(Dex) — 알파 헌터
  → 수는 적지만 LTV 최고
  → Oracle 리더보드의 핵심 콘텐츠 생산자
  → 이들의 참여가 플랫폼 신뢰도를 만듦
```

### Value Proposition (10초 버전)

**진(Jin)에게:**
> "내 트레이딩 데이터로 훈련되는 AI 에이전트.
> 써볼수록 나를 닮아가고, 내 약점을 보완해준다.
> 3Commas처럼 블랙박스가 아니라, 내가 직접 키운 AI."

**미나(Mina)에게:**
> "AI 오리를 키우면서 크립토 트레이딩을 배우는 게임.
> 차트 몰라도 괜찮아 — 오리가 알려줘.
> 놀수록 오리가 똑똑해지고, 그게 진짜 AI야."

**덱스(Dex)에게:**
> "네 판단으로 파인튜닝된 소형 모델. 네 것.
> 남들과 다른 데이터, 다른 모델, 다른 엣지.
> 성능은 Arena에서 증명되고, 네 트랙레코드가 된다."

### 런치 모션: Community-Led + Product-Led 하이브리드

```
Phase 0 (Pre-Launch, -8주):
  CLG 중심 — Twitter/Discord 커뮤니티 시드
  "나만의 트레이딩 AI를 키워보세요" 내러티브

Phase 1 (Soft Launch, -4주~0):
  PLG 시작 — 클로즈드 베타 (진 페르소나 200명)
  핵심 루프 검증: "게임 → 모델 개선 → 성능 증명"

Phase 2 (Public Launch):
  PLG + CLG — 공개 출시
  미나 유입: 다마고치 바이럴 + 소셜 공유
  덱스 유입: Oracle 리더보드 공개

## 5. 기능 재정의 (Game Loop 중심)

### 핵심 원칙: 게임플레이 = 모델 훈련

```
모든 유저 행동은 두 가지를 동시에 해야 한다:
1. 게임으로서 재미있어야 한다 (즉각 피드백, 보상)
2. 훈련 데이터로서 가치있어야 한다 (모델 개선에 기여)
```

### Micro Loop (1~3분) — "시그널 콜"

```
에이전트가 시그널 감지
  "BTC 4H — Double Bottom 감지. Confidence 72%"
      ↓
플레이어 판단 (15초)
  [LONG 👆]  [SHORT 👇]  [SKIP ⏭]
      ↓
결과 (30초~1분 후 실시간 가격으로 판정)
  맞음: 에이전트 🎉 + XP + Confidence ↑
  틀림: 에이전트 😔 + "왜 틀렸는지" 리뷰 카드
      ↓
[훈련 데이터 생성]
  {pattern: "double_bottom", timeframe: "4h",
   player_call: "long", outcome: "correct",
   market_context: {oi: ..., funding: ..., volume: ...}}
      ↓
다음 시그널 대기 (1~2분) → 반복
```

**이게 모델 훈련이 되는 이유:**
- 패턴별 플레이어 판단 정확도 축적
- 시장 컨텍스트(OI, 펀딩, 볼륨)와 결과의 상관관계
- ORPO: 정답 판단 = chosen, 오답 판단 = rejected → preference pair 생성

### Core Loop (15~30분) — "Arena 세션"

```
[Quick Scan] Terminal에서 시그널 콜 5~10개 (Micro Loop 반복)
  → 에이전트 Focus ↑, 패턴 데이터 축적
      ↓
[Draft] 3명의 AI 에이전트 + 전략 구성 (2분)
  → 훈련 데이터: squad composition preference
      ↓
[Battle] 5페이즈 매치 — 실시간 BTC 데이터 (10분)
  Draft → Analysis → Hypothesis → Battle → Result
  → 가장 풍부한 훈련 데이터 생성 구간
  → 방향 판단 + 리스크 설정 + 컨피던스 + 결과
      ↓
[Review] 에이전트 반응 + 성과 리뷰 (2분)
  맞으면: 에이전트 진화 포인트 ↑ + Trust ↑
  틀리면: "왜 틀렸는지" 분석 카드 + 패턴 약점 노출
  → 훈련 데이터: post-match reflection
      ↓
[Grow] 에이전트 상태 업데이트 확인
  모델 정확도 변화, 패턴 메모리 갱신, 진화 진행도
  → "다음 판 하자" 또는 "전략 조정하러 가자"
```

### Meta Loop (일~주) — "에이전트 성장"

```
Daily:
  - 에이전트 상태 확인 (Energy/Mood/Focus/Trust)
  - Morning Scan (시그널 콜 3~5개)
  - Arena 매치 1회 이상
  → 데일리 미션 완료 → 진화 포인트

Weekly:
  - 주간 성과 리포트 (모델 정확도 변화 그래프)
  - Oracle 랭킹 업데이트
  - 에이전트 진화 체크포인트
  → 실제 모델 파인튜닝 트리거 (주 1회 배치)

Monthly:
  - 시즌 랭킹 결산
  - Tier 변동 (Bronze → Silver → Gold → Diamond → Master)
  - 모델 성능 마일스톤 (정확도 50% → 55% → 60%)
  → 시즌 보상 + 커뮤니티 인정
```

## 6. 에이전트 시스템 재정의

### 에이전트 = 소형 모델 + 캐릭터

```
Technical Layer (소형 모델):
  - Base: 사전 훈련된 시장 분석 모델 (모든 유저 공통)
  - Personal: 유저 데이터로 파인튜닝된 레이어 (유저마다 다름)
  - Memory: 패턴별 승률, 레짐별 적응, 매치업 경험 (RAG)

Character Layer (다마고치):
  - 외형: Shell (Duckee/Dino/Nutty) + 성장 단계별 진화
  - 성격: 성장 경로에 따른 대사/반응 스타일
  - 상태: Energy / Mood / Focus / Trust (시간에 따라 변동)
  - 반응: 모든 유저 행동에 감정적 반응
```

### 에이전트 상태 시스템

| 상태 | 감소 조건 | 증가 조건 | Arena 영향 |
|------|----------|----------|-----------|
| **Energy** | 시간 경과 (-5%/hr), Arena 매치 (-15%) | 휴식(Dream 모드) (+30%), 로그인 보너스 (+10) | 낮으면 분석 속도 ↓ |
| **Mood** | 연패 (-10/패), 장시간 방치 (-5%/6hr) | 승리 (+10), 대화 (+15), 시그널 적중 (+5) | 낮으면 Confidence ↓ |
| **Focus** | 과다 사용 (-3/매치 after 5매치) | Terminal 스캔 (+10), 패턴 발견 (+15) | 높으면 패턴 감지율 ↑ |
| **Trust** | 급감 없음 (서서히만 변동) | 승리 (+5), 패배에도 (+2), 연승 보너스 (+3) | 높으면 결정적 순간 정확도 ↑ |

**핵심:** 상태가 Arena 성능에 **실제로** 영향. 돌봄이 곧 경쟁력.

### 에이전트 진화 (5단계)

| Stage | 조건 | 외형 변화 | 모델 변화 |
|-------|------|----------|----------|
| **Egg** | 생성 직후 | 기본 스프라이트 | Base 모델만 |
| **Baby** | 매치 5회 | 눈 표정 분화 | 첫 personal 레이어 생성 |
| **Teen** | 매치 20회 + Trust 50+ | 성장 경로별 악세사리 | 패턴 메모리 활성화 |
| **Adult** | 매치 50회 + Trust 80+ + Gold | 완전 진화 (경로별 다른 모습) | 레짐 적응 활성화 |
| **Legend** | Diamond + 특수 조건 | 희귀 이펙트 | 풀 ORPO 파인튜닝 모델 |

  - 커뮤니티 시그널 (다른 유저들의 시그널 콜)
  - Arena 결과 하이라이트 ("XX의 Duckee가 7연승!")
  - Copy Signal → Terminal에서 같은 시그널 콜 가능

[Oracle 탭]

  - 승률 기준 랭킹
