## **3.1 카카오벤처스 투자 판단 기준 적용**

카카오벤처스(브라이언 / 마이크)가 제시한 핵심 기준: (1) 문제가 진짜 손익 문제인가, (2) 만드는 것 이상으로 파는 것이 중요한가, (3) 속도가 생명인가, (4) 에이전트에게 선택받는가.

| 기준 | 판정 | 근거 |
| :---- | :---- | :---- |
| 문제가 손익 문제인가 | ✅ YES | 패턴 놓침 \= 실제 매매 손실. '기억 휘발'은 비용. 정량화 가능. |
| 파는 것이 중요한가 | ⚠️ 미검증 | Day-1 제품(challenge 저장+평가)이 유저에게 돈 낼 이유를 충분히 주는지 아직 불명확. |
| 속도가 생명인가 | ✅ YES | Minara 피벗 전에 Ledger를 쌓아야 함. MCL이 빠를수록 유리. |
| 에이전트에게 선택받는가 | ✅ 잠재 | MCP tool 4개 설계 완료. Pattern feed API → B2A 확장 논리 명확. |
| AI가 코어 가치가 아닌가 | ✅ YES | AI는 판단 보조. 코어 가치 \= Pattern Object \+ Ledger 누적. 대체 불가. |

## **3.2 YC 2025\~2026 투자 패턴 대조**

YC W2026 투자 패턴 5가지와 Cogochi를 대조한다.

### **Return Signals (YC W2026)와의 비교**

Return Signals: 반품 의도 고객을 교환으로 전환. 마진 5\~10% 개선, ROI가 즉시 계산된다.

Cogochi 대응: 패턴 놓침 → 매매 손실 방지. ROI \= '이 알림으로 진입했을 때 수익'. Ledger가 누적되면 정확히 계산 가능해진다.

* **유사점:** 좁은 버티컬 문제, AI가 코어가 아닌 엔진

* **차이점:** Return Signals는 즉시 ROI가 보임. Cogochi는 Ledger 50개+ 이후에야 ROI 증명 가능

### **Userlens (YC P2026)와의 비교**

Userlens: SaaS 고객 이탈 예측 \+ CSM 자동화. 여러 신호를 통합해 '지금 이 계정이 위험한지' 판단.

Cogochi 대응: 여러 파생상품 신호(OI, Funding, Volume, CVD)를 통합해 '지금 이 패턴이 먹힐지' 판단. 구조 동일.

* **유사점:** 다중 신호 통합 → 단일 판단. 데이터 연결이 해자. 끊으면 다시 수동으로 못 돌아감

* **차이점:** Userlens는 엔터프라이즈 B2B. Cogochi는 개인 트레이더 B2C → B2A

### **Bizzy AI (YC S2025)와의 비교**

Bizzy: AI 리셉셔니스트. SMB 운영 자동화. 빠른 매출화 가능하지만 경쟁 심화.

Cogochi 위험: 단순 알림 서비스로만 포지셔닝하면 Bizzy형 함정. 경쟁 진입 쉬움.

* **핵심 차이:** Cogochi의 방어선은 '알림'이 아니라 'Verdict Ledger 누적'. 알림 자체는 commodity.

## **4.2 Green / Yellow / Red 신호**

### **Green — 투자 근거가 되는 것**

* **Pattern Object \+ Phase State Machine:** 룰 기반이라 재현 가능하고 백테스트 신뢰 가능

* **Verdict Ledger:** 시간이 방어선. 6개월 후 500개+ 기록은 돈으로 살 수 없음

* **Refinement 구조:** 유저마다 다른 임계값 → 이탈 비용 발생

* **MCP tool 설계:** B2A 확장 논리 명확. 에이전트가 pattern feed를 읽는 구조

### **Yellow — 조건부 가능성**

* **LightGBM P(win):** 강력하지만 Ledger 50개 이전에는 아무 역할을 못 함

* **AutoResearch:** 내부 엔진으로는 중요, 카피로 전면에 내세우면 'AI 알아서 해줌' 기대 생성 위험

* **Bloomberg Terminal UI:** 진입 장벽이기도 하고 사용성 리스크이기도 함

### **Red — 지금 하면 안 되는 것**

* **KTO/LoRA per-user finetuning:** H1 검증 전까지 마케팅에 쓰면 안 됨

* **'AI가 매매해준다' 메시지:** commodity \+ 규제 리스크

* **캐릭터/세계관 레이어 (DOUNI):** 코어 통증과 멀어짐

## **5.3 수익화가 시작되는 조건**

**유저가 돈을 내는 이유는 그래프나 위키가 아니다. '내 패턴이 또 나왔다는 알림' \+ '그게 실제로 맞았는지 기록'이 반복될 때다.**

즉 수익화의 조건:

* State Machine이 실제로 알림을 보낸다 (MCL 완성)

* 유저가 알림 클릭 후 chart를 보고 ✓/✗ 체크한다 (리텐션)

* Verdict가 쌓이면서 hit rate가 보인다 (가치 증명)

* 그 hit rate가 $19 이상의 가치를 준다고 느낀다 (WTP 검증)

## **6.1 Kill Criteria (공개 기준)**

아래 조건 충족 시 즉시 방향 재설계:

* **H1 FAIL:** Ledger 50개 후 LightGBM AUC \< 0.55 → 패턴 정의 또는 feature 재설계

* **FEEDBACK DRY:** 2주 내 verdict 50개 미만 → 스캐너 정밀도 / UX 재검토

* **Alert CTR \< 30%:** M3 시점 알림 클릭률 30% 미만 → 스캐너 재설계

* **WAA \< 140:** M3 주간 활성 유저 140명 미만 → 타깃 페르소나 재검토

# **7\. 투자자용 요약 — 핵심 질문 답변**

## **Q1: AI를 썼다는 게 아니라 AI 없이는 안 나오는 경제성이 있나?**

**Yes. LightGBM P(win) \= 300개 심볼 × 92개 feature × 실시간 Phase 전환 시점을 동시에 처리하는 판단. 인간이 수동으로 할 수 없는 병렬성. 단, 이 주장은 Ledger 누적 후에야 수치로 증명된다.**

## **Q2: 데이터 해자가 진짜인가?**

**VerdictRecord \= features\_at\_entry(92개) \+ verdict \+ btc\_trend \+ user\_override. 이 조합은 외부에서 살 수 없다. 6개월 운영 후 패턴별 BTC 레짐 조건부 hit rate는 어떤 경쟁자도 갖지 못한 자산이 된다.**

## **Q3: 지금 제품은 팔리나?**

**아직 검증 안 됨. Day-1 \= challenge 저장 \+ 백테스트. 유저가 이것만으로 $19를 낼지는 미확인. MCL 완성(알림 \+ verdict) 이후가 진짜 WTP 검증 시점이다.**

## **Q4: 왜 지금이 시점인가?**

* 2026년 현재 per-user 패턴 학습 기능을 상용화한 크립토 툴 없음

* 파생상품 데이터(OI \+ Funding \+ CVD) API 접근성이 충분히 높아진 시점

* LLM 비용 하락으로 AutoResearch 자연어 파싱 비용 미미

* MCP 생태계 성숙 → B2A 확장 경로 명확
