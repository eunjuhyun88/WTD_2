# COGOCHI — `/lab` Lab 페이지

> 역할: 패턴 평가 + 검사 + 반복  
> 핵심: "내가 만든 패턴이 실제로 돈을 버는가?" — 숫자로 확인하는 공간  
> Challenge를 실행하고 SCORE, 승률, 기댓값을 확인한다

### 목록 항목 상태 표시

| 상태 | 표시 | 의미 |
|------|------|------|
| `⭐` | 별표 | 직접 상단 고정한 항목 |
| `⦿` | 기본 | 일반 항목 |
| 초록 SCORE | +0.0234 | 양의 기댓값 (Stage 1 통과 가능) |
| 빨간 SCORE | -1.000 | 음의 기댓값 (실패) |
| `미실행` | 회색 | 한 번도 평가 안 됨 |
| `❌ FAIL` | 빨간 배지 | Stage 1 게이트 미통과 |
| `✅ PASS` | 초록 배지 | Stage 1 게이트 통과 |
| `⚙ 실행 중` | 로딩 | 현재 평가 진행 중 |

### 동작

- 항목 클릭 → Right Pane에 해당 Challenge 상세 로드
- 별표(`⭐`) 클릭 → 상단 고정/해제 (로컬 저장)
- `[+ New Challenge]` → `/terminal` 이동 (새 패턴 만들기)
- 검색: slug 이름으로 실시간 필터
- 정렬: 최신순 / SCORE 높은순 / 이름순
- 필터: 전체 / PASS만 / FAIL만 / 미실행

## SCORE 카드 — 판정 기준 및 메시지

| SCORE | Stage 1 결과 | 메시지 |
|-------|-------------|--------|
| > 0, 6조건 모두 통과 | ✅ PASS | "엣지 있음. 실시간 시그널러 투입 가능." |
| > 0, 일부 조건 미통과 | ⚠ 부분 | "기댓값은 양수지만 조건 미달. 파라미터 조정 필요." |
| ≤ 0 | ❌ FAIL | "엣지 없음. Graveyard로 이동합니다." |
| N < 30 | ⚠ 데이터 부족 | "인스턴스 30개 미만. 유의미한 판정 불가." |

### Graveyard 처리
- FAIL 판정 Challenge는 목록에서 회색 처리 + `❌` 배지
- `[Graveyard 보기]` 버튼으로 필터 → 실패한 패턴 모아보기 가능
- 완전 삭제는 `[🗑 삭제]` 수동 클릭 필요

## 메트릭 상세 설명

| 메트릭 | 계산식 | 의미 |
|--------|--------|------|
| SCORE | expectancy × coverage | 종합 점수 |
| EXPECTANCY | mean(realized_pnl_pct) | 트레이드당 평균 PnL |
| WIN_RATE | count(r > 0) / n | 이긴 비율 |
| N_INSTANCES | 매칭된 진입 수 | 통계적 유의성 기반 |
| RISK_REWARD | abs(avg_win / avg_loss) | 평균 수익 / 평균 손실 비율 |
| PROFIT_FACTOR | sum(wins) / abs(sum(losses)) | 총수익 / 총손실 |
| MAX_DRAWDOWN | min(equity - peak) / peak | 최대 낙폭 |
| KELLY_FRACTION | Kelly 공식 | 최적 포지션 사이즈 (음수 = 투자 금지) |
| COVERAGE | 매칭 종목 수 / 전체 종목 수 | 범용성 |
| SORTINO | mean(r) / sqrt(mean(r² for r < 0)) | 하방 리스크 조정 수익 |
| TAIL_RATIO | p95 / abs(p5) | 극단 수익/손실 비율 |

## Instances 테이블 상세

### 열 구성

| 열 | 내용 |
|----|------|
| 심볼 | BTCUSDT, ETHUSDT 등 |
| 날짜 | YYYY-MM-DD HH:mm |
| 진입가 | 진입 시점 open 가격 |
| 실현 PnL | % 단위 (양수=수익, 음수=손실) |
| 종료 이유 | `target` / `stop` / `timeout` |
| 경과봉 | 종료까지 걸린 봉 수 |
| 액션 | `[차트 보기 →]` |

### 동작

- 행 클릭 또는 `[차트 보기 →]` → `/terminal?symbol=BTC&ts=2024-03-22T10:00` 딥링크
- Terminal Right Pane에 해당 시점 차트 로드 + 진입/stop/target 마커 표시
- PnL 열: 양수=초록, 음수=빨강
- 정렬: PnL 순 / 날짜순 / 심볼순
- 필터: 전체 / target만 / stop만 / timeout만

## Challenge 유형 2개

### 1. Pattern Hunting (손수 블록 조합)

- 블록 4종 구성: trigger + confirmation + entry + disqualifier
- 해석 가능, 직관적
- Terminal에서 자연어로 만들어서 저장

### 2. Classifier Training (LightGBM)

- 28개 feature → P(win) 확률 자동 학습
- 상세 설정:
  ```
  학습 종목:  [BTC ▼] [ETH ▼] [SOL ▼] ...
  학습 기간:  [6년 전체 / 최근 2년 / 커스텀]
  P(win) threshold:  [0.55 ▼]
  ```
- 결과: P(win) 히스토그램, Feature Importance 차트 (SHAP)
- Phase 1에서는 btc-macd-style 같은 수동 패턴이 우선
