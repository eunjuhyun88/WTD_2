## Part 1.5: 이론적 근거 — 5개 프레임워크

### F1. Hook Model (Nir Eyal, 2014)
**Trigger → Action → Variable Reward → Investment**

| 단계 | CHATBATTLE 적용 |
|------|----------------|
| **Trigger** | 외부: 푸시 알림 "BTC 5% 급락, 에이전트 판단: 매수 대기" / 내부: "내 수익 어떻게 됐지?" (감정) |
| **Action** | Home 열기 → 수익 숫자 확인 (5초 이내, 최소 마찰) |
| **Variable Reward** | 배틀 결과가 매번 다름 (승/패, 시나리오마다 다른 수익률). 도파민은 '결과 확인' 순간에 터짐 |
| **Investment** | 독트린 조정 시간, 메모리 카드 수집, 에이전트 레벨 → 떠나기 어려워짐 (매몰 비용) |

**핵심 설계**: 정우(Actor A)의 Hook 루프는 **"수익 확인"**이 Internal Trigger.
매일 아침 "어제 수익이 얼마였지?"라는 내적 충동 → Home → 숫자 확인 = 습관 형성.

### F2. Fogg Behavior Model (BJ Fogg, Stanford, B=MAP)
**Behavior = Motivation × Ability × Prompt**

각 페르소나별 첫 행동의 B=MAP 분석:

| 페르소나 | Motivation | Ability | Prompt |
|----------|-----------|---------|--------|
| 정우 | 높음 (돈 잃고 있음, 절실) | 중간 (개발자, 기술 OK) | 구글 검색 결과 |
| 수빈 | 중간 (돈은 벌고 싶지만 무서움) | 낮음 (코인 초보, 지갑 뭔지 모름) | 동료 추천 링크 |
| 태민 | 높음 (전략 수익화 욕구) | 높음 (Pine Script 쓰는 수준) | 커뮤니티 소문 |

**설계 결론**:
- 수빈을 잡으려면 **Ability를 극한까지 높여야** (온보딩 마찰 최소화). Motivation이 중간이므로 마찰 있으면 바로 이탈
- 정우는 Motivation이 높아서 약간의 마찰은 참음. 대신 **첫 Variable Reward를 빨리 보여줘야** (첫 배틀 승리 → 수익 숫자)
- 태민은 M+A 둘 다 높아서 Prompt만 있으면 됨. 대신 **"이건 트레이딩뷰보다 낫다"를 증명**해야 함

### F3. Jobs to be Done (Clayton Christensen, 2003)
"사람들은 제품을 '고용'해서 삶의 특정 '일'을 해결한다"

| 페르소나 | Functional Job | Emotional Job | Social Job |
|----------|---------------|---------------|------------|
| 정우 | "감정 개입 없이 매매하기" | "나도 수익 내는 트레이더가 되고 싶다" | "친구들한테 '나 AI로 매매해' 자랑하고 싶다" |
| 수빈 | "검증된 전략을 자동으로 따라하기" | "사기 안 당하고 안전하게 돈 벌고 싶다" | "동료한테 '나도 코인 하는데 플러스야' 말하고 싶다" |
| 태민 | "내 전략의 과적합 여부를 검증하기" | "내가 진짜 잘한다는 걸 증명하고 싶다" | "구독자 100명 = 퀀트 트레이더로서의 사회적 인정" |

**설계 결론**: 각 페이지는 Functional Job을 수행하되, Emotional + Social Job을 UI로 강화해야 한다.
예: Lab에서 "승률 73%"는 Functional. "상위 5% 전략" 배지는 Emotional+Social.

### F4. Variable Ratio Reinforcement (B.F. Skinner, 1950s)
"불확실한 보상 = 가장 강력한 행동 유지 장치"

도파민은 보상 자체가 아니라 **보상의 불확실성**에 반응 (Zald et al., 2004).

CHATBATTLE의 Variable Reward 시스템:
- **배틀 결과**: 승/패 + 수익률이 매번 다름 (같은 전략이라도 시나리오에 따라 변동)
- **메모리 카드**: 어떤 ERA에서 배틀하느냐에 따라 다른 카드 획득 (수집 욕구)
- **시장 스캔**: 5-Agent 판단이 매번 다름 (같은 BTC라도 타이밍에 따라 LONG/SHORT 갈림)
- **구독자 수익**: "오늘 구독자가 늘었나?" (태민의 Variable Reward)

**핵심 설계**: Battle 결과 화면에 **지연 공개** 넣기 (드럼롤 → 결과). 즉시 보여주면 안 됨.
"불확실성 순간"이 도파민 피크. 이걸 UX로 설계해야 함.

### F5. 수요-공급 구조 — 솔직한 분석

**내가 이 사업을 한다고 가정하고 냉정하게 본 문제들:**

#### 근본 문제: "시뮬레이션으로는 돈을 못 번다"

현재 CHATBATTLE의 배틀 = 역사적 데이터 시뮬레이션.
정우는 진짜 돈을 벌러 왔는데, 시뮬레이션 승리는 돈이 아니다.

```
정우가 원하는 것:  바이낸스 잔고 +$127  (진짜 돈)
CHATBATTLE이 주는 것:  시뮬레이션 승률 68%  (가짜 수치)
→ 이 간극이 안 메워지면 Day7에 앱 삭제
```

수빈도 마찬가지:
- Bitget 카피 트레이딩: **무료**, 실제 돈 자동 매매
- CHATBATTLE: **$15/월**, 시뮬레이션 기반 → 왜 여기 쓰냐?

태민도 마찬가지:
- TradingView: 유저 수백만, Pine Script 마켓플레이스 이미 존재
- CHATBATTLE: 유저 0명 → 왜 여기 퍼블리시하냐?

#### 현재 설계의 치명적 문제 5가지

**① 실제 돈과의 연결이 없다**
- "배틀에서 이겼다" ≠ "돈을 벌었다"
- 게임 포인트/메모리 카드/레벨 = 유저가 원하는 것이 아님
- 유저가 원하는 것: **바이낸스 잔고 증가**

**② 경쟁사 대비 뚜렷한 이유가 없다**
- 정우: "3Commas $14.5/월로 실제 자동매매 되는데, 왜 여기서 시뮬레이션을?"
- 수빈: "Bitget 카피 트레이딩 무료인데, 왜 여기서 $15 내고 시뮬레이션을?"
- 태민: "TradingView에 유저 6천만명인데, 왜 유저 없는 여기서?"

**③ 3가지를 동시에 하려 한다**
- AI 트레이딩 봇 (정우용) + 카피 트레이딩 마켓플레이스 (수빈용) + 전략 검증 도구 (태민용)
- 세 개를 다 하면 → 어느 것도 제대로 못 함
- 스타트업이 해야 할 것: **하나만 제대로**

**④ 게임 레이어가 비용이지 가치가 아니다**
- 배틀, 메모리 카드, 에이전트 레벨 = 개발 비용 큼
- 정우: "게임 하러 온 게 아니라 돈 벌러 왔는데"
- 게임화는 핵심 가치가 작동한 뒤에 리텐션 목적으로 추가해야 함
- **핵심 가치 전에 게임화 = 장식에 돈 쓰는 것**

**⑤ 수익 모델이 안 맞다**
- $51K/월 추정 → Builder 200명 + Copier 2,000명 + Trader 500명 = 비현실적
- 크립토 앱 Day30 리텐션: 2-3%
- 2,700명 활성 유저 유지하려면 → MAU 기준 13.5만명 획득 필요
- 현실적으로 1-2년 내 불가능

#### 진짜 사업이 되려면? — 재설계 방향

**원칙: 하나만. 실제 돈. 오늘부터.**

**가장 강한 자산**: Agent Forge의 5-Agent 파이프라인 + forward-walk 검증
**가장 약한 부분**: 시뮬레이션 ↔ 실매매 연결

**방향 A: "AI 트레이딩 시그널" (정우 퍼스트)**
```
핵심 가치: 5-Agent AI가 BTC 분석 → "지금 LONG, 신뢰도 78%" 시그널
실제 돈 연결: 바이낸스 API → 원클릭 실행 or 자동 실행
수익: $29/월 구독 or 수익의 20% 성과보수
경쟁자: 3Commas, Stoic.ai, Cryptohopper
차별점: 5-Agent 합의 시스템 (단일 봇이 아닌 위원회 판단)
검증: forward-walk → "이 시그널은 200일 검증됨"

장점: 바로 돈이 됨, 단일 유저 타입
단점: 거래소 API 연동 개발 필요, 규제 리스크
```

**방향 B: "검증된 카피 트레이딩" (수빈 퍼스트)**
```
핵심 가치: 실매매 기록이 검증된 트레이더만 카피 가능
실제 돈 연결: Bitget/Binance 카피 트레이딩 API 위에 검증 레이어
수익: 카피 수익의 10% 성과보수
경쟁자: Bitget (검증 없음), eToro (검증 약함)
차별점: forward-walk + 온체인 트랙레코드 → 조작 불가

장점: Copier 시장이 큼 ($4.27B), 검증이 진짜 차별점
단점: 먼저 좋은 트레이더를 확보해야 함 (Cold Start)
```

**방향 C: "전략 검증 SaaS" (태민 퍼스트)**
```
핵심 가치: Pine Script → 업로드 → 자동 백테스트 + forward-walk + 리포트
실제 돈 연결: 검증 결과로 투자 판단, 마켓플레이스 퍼블리시
수익: Pro 검증 $29/월, 기업 라이선스 $299/월
경쟁자: TradingView (검증 약함), QuantConnect (복잡함)
차별점: "원클릭 forward-walk" = TradingView에서 못 하는 것

장점: Cold Start 없음 (Single-Player 가치), 기술 해자
단점: 시장이 작음 (퀀트 트레이더만), 확장성 제한
```

**내 추천: 방향 A → B → C 순서**

```
Month 1-3:  방향 A — AI 시그널 (Terminal + 거래소 연동)
            정우 같은 유저 100명만 확보
            바이낸스 API 연결 → "진짜 돈"이 움직이기 시작

Month 4-6:  방향 A에서 성과 좋은 유저의 기록이 쌓임
            → 자연스럽게 방향 B로 확장
            "이 사람 6개월 +42% 검증됨. 따라할래?"
            = Cold Start 해결 (A의 유저가 B의 공급자)

Month 7+:   방향 C 추가
            태민 같은 빌더에게 전략 검증 도구 제공
            마켓플레이스 완성
```

**게임 레이어 (배틀, 메모리 카드) = Month 7+ 이후**
핵심 가치가 작동한 뒤 리텐션 메커니즘으로 추가.
지금은 불필요.

**핵심 원리: 유저가 돈을 벌어야 플랫폼이 번다**

시뮬레이션은 "게임"이 아니라 **훈련**이다.
훈련의 목적은 **실전 배치 → 유저가 실제로 돈을 번다**.
유저가 돈을 벌면 → 더 많이 쓰고 → 플랫폼이 더 번다.

```
[훈련 단계]                           [실전 단계]
시뮬레이션 배틀 → 에이전트 성장 →      바이낸스 실매매
백테스트/Lab  → 전략 검증 →           거래 수수료 발생
독트린 최적화 → 승률 증명 →           유저 수익 발생
                                     ↓
                              유저가 돈을 번다
                                     ↓
                              더 키우고 싶다 (크레딧 구매)
                              다른 거래소도 연결 (추가 수수료)
                              새 에이전트 만들기 (슬롯 구매)
                              전략 퍼블리시 (마켓플레이스 수익)
```

**가치 순환 구조:**
```
유저 돈 벌기 ←──── 에이전트 실전 배치
     │                    ↑
     ↓              에이전트 성장 (시뮬레이션)
더 투자하고 싶다          ↑
     │              크레딧/구독으로 더 훈련
     └──────────→ 플랫폼 수익
```

이게 되면:
1. **무료 → 크레딧 전환**: "이거 실매매에 쓰면 진짜 돈 되겠다" → 더 훈련하고 싶어짐
2. **구독 전환**: 매일 훈련하려면 크레딧보다 구독이 싸다
3. **실매매 활성화**: 증명된 에이전트로 바이낸스에서 실제 매매 → 유저가 돈을 번다
4. **거래 수수료**: 유저가 벌수록 플랫폼도 번다 (인센티브 정렬)
5. **확장**: "이 에이전트를 Bybit에서도 쓸래" → 추가 연동 수수료
6. **마켓플레이스**: "내가 키운 에이전트 전략을 팔래" → 구독자 수익의 20%

**왜 이 모델이 되는가:**
- 이전: "게임에 돈 써" → 유저 반응: "게임은 무료여야지"
- 이제: "훈련에 돈 써 → 그 결과로 실제 돈 벌어" → 유저 반응: "투자 대비 수익이 있네"
- **유저의 과금이 투자처럼 느껴진다** = 가장 강한 과금 동기

**이전 모델 vs 이 모델:**
- 이전: "시그널 구독" → 유저가 "왜 여기서?" → 경쟁사 대비 이유 부족
- 이제: "내 AI 캐릭터 키워서 실전에서 돈 벌기" → **경쟁사에 없는 경험**
- 핵심: **시뮬레이션이 훈련이고, 실매매가 졸업. 유저가 진짜 돈을 벌어야 성공.**

#### F5-d. 최종 비전 — "AI 에이전트 키우기 = 제품"

**3가지 레이어가 하나로 합쳐진 경험:**

```
┌─ 타마고치 레이어 (감정/애착) ────────────────────────┐
│ • 내 에이전트가 매일 자라고 있다                       │
│ • 오늘은 뭘 먹였지? (데이터 피딩)                      │
│ • 기분이 어때? (시장 상태 반영)                         │
│ • 성장 확인하는 게 매일의 루틴                          │
│ → 감정적 유대 = "이 에이전트는 내가 키운 거야"          │
└─────────────────────────────────────────────────────┘

┌─ RPG 레이어 (전략/성장) ─────────────────────────────┐
│ • 5개 에이전트 각각 스탯 (정확도, 리스크 관리, 타이밍)   │
│ • 독트린 = 장비. 교체하면 전략이 바뀜                   │
│ • 시나리오 = 던전. 2020 블랙서스데이, 2024 반감기 등    │
│ • 레벨업 = 승률 누적. 팀 구성이 결과에 영향             │
│ → 전략적 깊이 = "이 조합이 이 시장에서 최강이야"        │
└─────────────────────────────────────────────────────┘

┌─ 연구소 레이어 (검증/과학) ──────────────────────────┐
│ • 백테스트 = 실험. 가설 → 검증 → 결과                  │
│ • forward-walk = 논문 리뷰. 과적합 아닌 증명            │
│ • Auto-Optimize = 자동화된 실험. 최적 파라미터 탐색      │
│ • 그래프와 숫자 = 과학적 리포트                         │
│ → 신뢰감 = "이건 느낌이 아니라 데이터로 증명된 거야"     │
└─────────────────────────────────────────────────────┘

         세 레이어가 합쳐지면:
         ↓
┌─ 졸업 = 실전 배치 ──────────────────────────────────┐
│ "내가 키운 + 전략 짠 + 검증된 에이전트를 실전에 투입"   │
│ → 바이낸스에서 실제 매매 → 실제 수익 발생               │
│ → 수익이 나면 → 더 키우고 싶다 → 돌아옴                │
│ → 수익이 안 나면 → 개선하러 → 돌아옴                   │
│ = 어떤 결과든 돌아온다                                 │
└─────────────────────────────────────────────────────┘
```

**각 레이어가 페이지에 어떻게 녹는가:**

| 페이지 | 타마고치 | RPG | 연구소 |
|--------|---------|-----|--------|
| HOME | "오늘 에이전트 상태" | 레벨/스탯 요약 | 실매매 PnL 숫자 |
| TERMINAL | "시장을 먹여줬다" | 5-Agent 분석 카드 | 팩터 점수 그래프 |
| LAB | "실험 시켜봤다" | 시나리오 던전 선택 | 백테스트 리포트 |
| ARENA | "배틀로 경험치 획득" | 팀 전투 + 결과 | MFE/MAE + GameRecord |
| AGENT | "성장 기록 + 애정 확인" | 스탯/레벨/장비 관리 | 트랙레코드 증명 |
| MARKET | "내 에이전트를 자랑" | 리더보드 | 검증 배지 + 수익률 |

**일상 루프 (세 레이어 통합):**
```
아침: HOME 열기 (타마고치)
  → "에이전트 오늘 컨디션: 시장 상승 추세, 자신감 높음"
  → "어제 실매매 +$47" (연구소 숫자)

점심: TERMINAL 스캔 (RPG + 연구소)
  → 5-Agent 분석 확인
  → "독트린 교체할까?" (RPG 장비 교체 느낌)

저녁: LAB에서 실험 하나 (연구소)
  → "이 파라미터 바꾸면 승률이 올라갈까?"
  → 백테스트 실행 → 결과 확인

(가끔): ARENA 배틀 (RPG + 타마고치)
  → 새 시나리오에서 에이전트 성장
  → 메모리 카드 획득 (수집 욕구)

(주1회): AGENT 점검 (전체)
  → 성장 그래프 확인
  → 실전 투입 결과 리뷰
  → 개선점 발견 → 다시 훈련
```

#### F5-e. 온체인 토큰 경제 — 토큰이 흐르는 곳 = 페이지 구조

**토큰: $CLAW (유틸리티 토큰)**

│ → $CLAW가 있어야 뭘 할 수 있음 = 토큰 수요             │
└───────────────────────────────────────────────────────┘

┌─ 토큰 SOURCE (획득 = 공급) ──────────────────────────┐
│                                                       │
│ [ARENA]   배틀 승리 보상        3-10 $CLAW             │
│ [ARENA]   연승 보너스           +50% 보상              │

│ [스테이킹] 장기 스테이킹 이자   APY 5-15%              │

└───────────────────────────────────────────────────────┘

**토큰 흐름이 만드는 선순환:**
```
돈 벌고 싶다 → $CLAW 구매 or 무료 배틀로 획득
     ↓
$CLAW로 에이전트 키움 (훈련, 백테스트, 최적화)
     ↓
에이전트 성장 → 실전 배치 ($CLAW 스테이킹)
     ↓
실매매 수익 발생 → 수익 일부가 $CLAW로
     ↓
$CLAW로 더 키우기 or 현금화
     ↓
더 키우면 → 더 잘 벌고 → 더 많은 $CLAW
= 토큰 보유 인센티브 + 에코시스템 성장
```

**토큰 소비 포인트가 페이지 구조에 미치는 영향:**

| 페이지 | 무료로 되는 것 | 토큰 필요한 것 | 과금 심리 |
|--------|-------------|-------------|----------|
| HOME | 기본 상태 확인 | - | "무료라 편해" |
| CREATE | - | 에이전트 민팅 100 $CLAW | "투자의 시작" |
| TERMINAL | 기본 스캔 3회/일 | 프리미엄 데이터, 무제한 스캔 | "더 좋은 데이터 = 더 좋은 판단" |
| LAB | 기본 백테스트 (50일) | 200일 백테스트, Auto-Optimize, forward-walk | "더 정확한 검증 = 더 확실한 실전" |
| ARENA | 기본 시나리오 3회/일 | 프리미엄 시나리오, 추가 배틀 | "더 다양한 경험 = 더 강한 에이전트" |
| AGENT | 기본 성장 확인 | 추가 슬롯, 실전 배치 | "실전 투입 = 진짜 돈 벌기의 시작" |

**핵심: 무료로 Hook → 토큰으로 성장 가속 → 실전 수익으로 토큰 회수**

유저가 토큰을 사는 이유: "이건 비용이 아니라 투자. 에이전트가 돈을 벌어오니까."

┌─ 온체인 (Base L2 or Solana) ─────────────────────┐

**온체인에 기록되는 것 (= 신뢰의 원천):**

**GAP 2: 에이전트 간 거래 (Agent-to-Agent Commerce)**

유저의 에이전트끼리 상호작용하는 3가지 모드:

```
Mode 1: Agent vs Agent 배틀
┌──────────────────────────────────────────────┐
│ 내 에이전트 vs 다른 유저의 에이전트              │
│ 같은 시나리오 → 누가 더 잘 판단하는가?          │
│ 승자: $CLAW 보상 + ELO 레이팅 상승             │
│ 패자: 경험치 획득 (학습 데이터)                  │
│                                              │
│ → PvP 배틀 = 가장 강한 리텐션 메커니즘          │
│ → "내 에이전트가 저 사람 거보다 잘한다" = 자랑    │
└──────────────────────────────────────────────┘

Mode 2: Agent 데이터 거래
┌──────────────────────────────────────────────┐
│ 에이전트 A가 발견한 패턴을 에이전트 B에게 판매    │
│ 예: "블랙서스데이 시나리오에서 배운 교훈"         │
│    = 메모리 카드를 $CLAW로 거래                 │
│                                              │
│ → 메모리 카드 마켓플레이스                      │
│ → 경험이 자산이 됨                             │
└──────────────────────────────────────────────┘

Mode 3: Agent 합성 (Fusion)
┌──────────────────────────────────────────────┐
│ 에이전트 A의 REGIME 스탯 + 에이전트 B의 FLOW 스탯│
│ = 새로운 하이브리드 에이전트 생성                │
│ 비용: 200 $CLAW + 양쪽 에이전트 NFT 번         │
│                                              │
│ → 포켓몬 교배처럼 — 최적 조합 찾기 = 깊은 전략성 │
│ → $CLAW burn = 디플레이션 압력                  │
└──────────────────────────────────────────────┘
```

**UX 영향:**

### F5-b. 행동편향 역이용 (Behavioral Bias Reversal)
논문 기반: Skwarek (2025, SAGE Open), Yachna & Walia (2025, IJARCMSS)

트레이더의 주요 편향과 CHATBATTLE의 대응:

| 편향 | 현실 | CHATBATTLE 역이용 |
|------|------|-------------------|
| **손실 회피** (Loss Aversion) | 손절 못 함 → 큰 손실 | AI가 자동 손절 → "인간이 못 하는 걸 AI가 해준다" = 핵심 가치 |
| **FOMO** (Fear of Missing Out) | 이미 오른 거 추매 → 물림 | 5-Agent 분석이 "진입 늦음" 경고 → FOMO 차단 = 차별점 |
| **과신 편향** (Overconfidence) | 연승 후 레버리지 올림 → 청산 | Lab에서 객관적 승률 표시 → 과신 방지 |
| **처분 효과** (Disposition Effect) | 수익은 빨리, 손실은 늦게 청산 | AI가 TP/SL 기계적으로 실행 → 처분 효과 제거 |
| **허드 멘탈리티** (Herding) | 남들 따라 매수 → 꼭지 매수 | 개별 에이전트의 독립적 판단 = 허드에서 이탈 |

**설계 결론**: Home에서 "AI가 오늘 당신 대신 3번 손절했습니다 (수동이었으면 -$230 손실)"
→ **"AI를 쓰지 않았으면 이만큼 잃었다"** 프레이밍 = 가장 강력한 리텐션 메시지

### 1. HOME `/`
**의도:** "오늘 뭐해야 해?" + 첫 방문자 전환

**첫 방문자 (미인증):**

**Builder (인증됨):**
- 데일리 허브 카드:
  - 오늘 배틀 할당: 2/5 [→ Battle]

  - 활성 시그널: 2개, +$42 [→ Signals]
  - 최근 스캔 요약: "BTC 상승 모멘텀" [→ Terminal]

**Copier (인증됨):**
- 포지션 대시보드:
  - 라이브 포지션 리스트 + PnL
  - 구독 에이전트 활동 타임라인
  - 빠른 액션: [새 에이전트 구독] [시그널 탐색]

  - 프리셋 파라미터 확인/수정
- Step 3: 민트 + AI 바인드

- Step 4: 튜토리얼 배틀로 이동
  - "첫 배틀을 시작하겠습니다!" → Battle (가이드 모드)

## Part 6: 자기 반박 — 발견된 논리적 결함 12개

### ❌ Flaw 1: 시뮬레이션 → 실전 브릿지가 논리적으로 안 맞음
시뮬레이션은 **이미 결과를 아는** 과거 데이터에서 돌림.
forward-walk "통과"는 **과거에 됐다**는 증명이지, **미래에도 된다**는 증명이 아님.
→ 유저에게 "이 전략은 검증됐습니다" 라고 말하는 건 과잉 약속일 수 있음.
→ **해결 필요**: 실매매 성과 추적 기간(예: 2주 페이퍼 트레이딩) 의무화? "과거 검증 + 실시간 검증" 2단계?

### ❌ Flaw 2: 두 개의 모순된 사업 모델이 공존
**F5**: "게임 레이어는 Month 7+ 이후. 지금은 불필요. 하나만 제대로."
**F5-d**: "타마고치 × RPG × 연구소 = 제품. 게임이 핵심."
→ 하나는 "게임 나중에" 다른 하나는 "게임이 핵심"
→ **해결 필요**: 둘 중 하나 선택. 아니면 Phase 1에서 게임 최소화 + Phase 2에서 확장이라는 명시적 로드맵

### ❌ Flaw 3: Fiat 결제(Stripe)와 $CLAW 토큰이 충돌
- Layer 2: Credits (Stripe → $5 → 100크레딧)
- $CLAW: 온체인 토큰 (ERC-20)
→ 유저가 뭘로 결제하는가? Stripe? 지갑? 둘 다?
→ Pro 구독($29/월)을 내면 $CLAW도 필요한가?
→ **해결 필요**: Fiat path vs Crypto path 명확히 분리, 또는 하나만 선택

### ❌ Flaw 4: PvP 배틀이 런칭 시 물리적으로 불가능
100명 유저로 PvP 매칭? 동시접속 5명이면 매칭 안 됨.
→ **해결 필요**: Phase 1은 PvE(AI 상대)만. PvP는 유저 1,000명 이후

### ❌ Flaw 5: "수동매매 43%" 비교 데이터가 없음
Lab 와이어프레임의 핵심: "당신(수동): 43% vs AI: 68%"
→ 유저의 과거 매매 데이터가 없으면 이 비교가 불가능
→ **해결 필요**: ① 거래소 API 연결 후 자동 임포트, 또는 ② "평균 크립토 트레이더" 벤치마크 사용

### ❌ Flaw 6: $CLAW 토큰 부트 패러독스
- 에이전트 민팅: 100 $CLAW 필요
- $CLAW 획득: 배틀 승리 3-10개
- 배틀하려면 에이전트 필요
→ 에이전트 없이는 $CLAW 못 벌고, $CLAW 없이는 에이전트 못 만듦
→ **해결 필요**: 첫 에이전트는 무료 (100 $CLAW 에어드랍 or 무료 민팅)

### ❌ Flaw 7: "무료 에이전트 1개" vs "민팅 100 $CLAW"
Layer 1(Free to Play): "에이전트 1개 생성"
토큰 Sink: "에이전트 민팅 100 $CLAW 번"
→ 무료인가 100 $CLAW인가?
→ **해결 필요**: 첫 에이전트 = 무료, 추가 에이전트 = 100 $CLAW

### ❌ Flaw 8: 졸업 게이트 67일은 정우를 죽임
forward-walk에 200전 필요 → 3회/일 무료 → 67일

## Part 7: 12개 결함 해결안

### ✅ Fix 1: 시뮬→실전 브릿지 — "3단계 검증 체계"

과거 검증만으로는 부족. **3단계 점진적 검증 + 리스크 공개:**

```
Stage A: 시뮬레이션 검증 (Lab)
  → forward-walk 통과 → "과거 데이터에서 유효"
  → 라벨: "시뮬 검증됨" (주황 배지)

Stage B: 페이퍼 트레이딩 (2주)
  → 실시간 시장에 가상 주문 → 실제 가격으로 PnL 계산
  → 실제 돈 안 움직이지만 실시간 데이터
  → 라벨: "실시간 검증됨" (파랑 배지)

Stage C: 실매매 (소액 → 점진적 확대)
  → 첫 2주: 최대 $100/트레이드
  → 다음 2주: 실매매 PnL 기반으로 한도 자동 조정
  → 라벨: "실전 검증됨" (초록 배지) + 기간 표시
```

**리스크 공개 의무**: 모든 검증 배지 옆에:
> "과거 성과는 미래 수익을 보장하지 않습니다. 최대 드로다운: -X%"

이렇게 하면:
- 정우: Stage B(페이퍼 2주) → Stage C 소액 → "진짜 돈" 2주 만에 가능
- 수빈: 구독 에이전트가 Stage C 배지 → "실전 검증" 신뢰
- 규제: "투자 조언"이 아닌 "도구 제공" + 리스크 고지

  - 리텐션 데이터 기반으로 어떤 게임 요소가 효과적인지 확인 후 투자

→ 모순 해소: **게임은 처음부터 있지만 MVP 수준, 실전 연결이 Day 1 가치, 게임은 리텐션 강화용으로 점진 확대**
