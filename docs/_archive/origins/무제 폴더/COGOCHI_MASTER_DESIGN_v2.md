## 0. 한 줄 정의

> **내 판단으로 훈련되는 나만의 트레이딩 AI. 쓸수록 나를 닮고, 나보다 나아진다.**

DOUNI는 펫이 아니다. DOUNI는 내 트레이딩 AI 모델의 얼굴이다.
키운다 = 훈련한다. 돌본다 = 데이터를 먹인다. 성장한다 = 모델이 좋아진다.

### 1.2 핵심 사업 원리

```
유저가 쓴다 → 훈련 데이터가 쌓인다 → 모델이 좋아진다
     ↑                                      ↓
     └────── 모델이 좋아지니 더 쓴다 ────────┘

이 순환이 만드는 것:
1. 전환 비용 (switching cost) — 3개월 훈련한 모델을 버리겠는가?
2. 네트워크 효과 — 유저가 많을수록 Base Model이 좋아짐
3. 데이터 해자 — 유저 판단 데이터는 우리만 가짐
```

### 1.3 North Star Metric (NSM)

**NSM: 주간 모델 정확도 향상 유저 수 (Weekly Users with Improving Accuracy)**

```
정의: 이번 주 에이전트 Accuracy가 지난 주보다 올라간 유저 수.

왜 이게 NSM인가:
┌─────────────────────────────────────────────────────────┐
│                                                          │
│  정확도 향상 → 유저가 "이거 진짜 작동하네" 체감          │
│       ↓                                                  │
│  계속 사용 → 더 많은 훈련 데이터 생성                    │
│       ↓                                                  │
│  모델이 더 좋아짐 → 유료 전환 / 업그레이드                │
│       ↓                                                  │
│  매출 증가 + 리텐션 증가 + 바이럴 ("내 AI 정확도 63%!")  │
│                                                          │
│  이 하나의 숫자가 올라가면 모든 게 따라온다.              │
└─────────────────────────────────────────────────────────┘

왜 다른 지표가 아닌가:
  MAU           → 허영 지표. 많아도 가치 체감 안 하면 의미 없음.
  MRR           → 결과 지표. 이걸 직접 올릴 수는 없음.
  세션 길이      → 오래 있는다고 가치는 아님. 헤매는 것일 수 있음.
  분석 횟수      → 많이 한다고 모델이 좋아지는 건 아님.

  "정확도 향상 유저 수"는:
  1. 제품 가치 전달을 직접 측정 (모델이 진짜 좋아지고 있는가?)
  2. 행동 가능 (어떤 기능이 정확도 향상에 기여하는가?)
  3. 선행 지표 (정확도 올라가면 → 유료 전환, 리텐션 따라옴)
```

**NSM 분해 (Input Metrics):**

```
NSM = f(활성 유저 × 판단 품질 × 학습 효율)

                 NSM
        주간 정확도 향상 유저 수
              /        \
             /          \
    활성 분석 유저      모델 학습 효율
    (Weekly Active      (Accuracy Δ
     Analysts)          per 100 pairs)
       /    \              /     \
      /      \            /       \
  세션 빈도  세션 깊이  데이터 품질  파이프라인
  (주 몇 회) (판단 몇 개) (pair 유효율) (학습 주기)
```

| 레벨 | 지표 | 목표 | 측정 |
|------|------|------|------|
| **L0 NSM** | 주간 정확도 향상 유저 수 | MAU의 40%+ | 주간 |
| **L1 활성** | Weekly Active Analysts | MAU의 60%+ | 주간 |
| **L1 효율** | Accuracy Δ per 100 pairs | +1%p 이상 | 월간 |
| **L2** | 주간 세션 빈도 | 3회+ | 주간 |
| **L2** | 세션당 판단 수 | 5개+ | 세션 |
| **L2** | 훈련 pair 유효율 | 70%+ | 월간 |
| **L2** | 모델 학습 주기 | 주 1회 | 운영 |

**각 팀이 보는 것:**

```
Product:  "어떤 기능이 세션당 판단 수를 늘리는가?"
ML:       "어떤 학습 방법이 Accuracy Δ를 높이는가?"
Growth:   "어떤 채널의 유저가 L1 활성률이 높은가?"
Revenue:  "정확도 향상 유저의 유료 전환율은?"
          (추정: 정확도 향상 체감 유저의 전환율이 3x 높을 것)
```

## 2. 제품 정의

### 2.1 무엇인가

트레이딩 전용 대화형 AI 플랫폼.
대화하면서 시장을 분석하고, 그 과정이 자동으로 개인 AI 모델을 훈련시킨다.
DOUNI는 그 모델의 시각적 인터페이스.

### 2.2 에이전트 = 모델

```
DOUNI ≠ 펫
DOUNI = 내 트레이딩 AI 모델의 시각화

DOUNI가 웃으면     = 모델 정확도가 올라가고 있다
DOUNI가 흥분하면   = 강한 시그널을 감지했다
DOUNI가 고민하면   = 데이터가 충돌한다 (확신 낮음)
DOUNI가 성장하면   = 모델 아키텍처가 업그레이드됐다

"키운다" = 훈련 데이터를 만든다
"돌본다" = 분석하고 피드백한다
"장비 장착" = feature set을 커스텀한다
"진화" = LoRA rank 증가 / 추론 스테이지 추가
```

### 2.3 3-Layer (v1의 4-Layer에서 축소)

```
Layer 1: 트레이딩 인텔리전스 (코어 가치)
  대화로 데이터 검색/분석 + AI 공동 판단 + 매매 실행
  → "이게 있으니 돈 낼 만하다"

Layer 2: 개인화 AI 모델 (전환 비용)
  내 판단으로 훈련된 per-user LoRA
  → "이걸 버릴 수 없다"

Layer 3: DOUNI 캐릭터 (감정 + 리텐션)
  모델 상태의 시각화 + 반응 + 성격
  → "이 AI가 좀 특별하게 느껴진다"
```

Layer 1이 없으면 아무도 안 쓴다. Layer 2가 없으면 떠난다. Layer 3가 없으면 차갑다.
순서가 중요하다. 가치 먼저, 잠금 다음, 감정 마지막.

COGOCHI가 대체하는 것:

```
TradingView  → COGOCHI 차트 + 인디케이터 (TradingView 임베드)
Glassnode    → COGOCHI 온체인 데이터 (대화로 검색)

5개 구독 = $150~300/월
COGOCHI Pro = $49/월
절약: $100~250/월
```

### 3.2 돈 내는 진짜 이유: "이건 내 AI다"

```
Month 1: "편하네. 말하면 차트 나오고 데이터 나오고."
         → 도구 가치. 여기까진 무료도 됨.

Month 2: "어? 내가 자주 보는 패턴을 DOUNI가 먼저 찾아주네."
         → 개인화 시작. "이거 좀 다른데?"

Month 3: "DOUNI가 제안한 숏 포지션, 내가 혼자 봤으면 놓쳤다."
         → 실질적 가치 체감. "이거 없으면 불편하겠다."

Month 4: "내 승률이 52%에서 58%로 올랐다. DOUNI 정확도도 같이 올랐다."
         → 데이터로 확인. "이건 진짜 작동하는 거다."

이 시점에서: "Pro로 올려야겠다. 더 많은 인디케이터랑 LoRA 커스텀 쓰고 싶다."
```

### 3.3 소유욕 = 전환 비용

```
내 DOUNI에 쌓인 것:
  • 3개월치 매매 판단 데이터 (1,200+ preference pairs)
  • 내 스타일에 맞춰진 LoRA weights
  • 패턴별 승률 기록 (Double Bottom에서 72% 승률)
  • 레짐별 적응 데이터 (하락장에서 이렇게 대응한다)
  • 내가 자주 보는 인디케이터 조합의 최적 가중치

이걸 버리고 떠나는 것 = 3개월을 버리는 것
경쟁사로 가면 = 0부터 다시 시작

→ Spotify 플레이리스트보다 강한 전환 비용
  (플레이리스트는 export 가능. 훈련된 모델은 불가.)
```

## 4. 에이전트 상태 = 모델 메트릭

### 4.1 v1 → v2 변경

```
v1 (펫 상태):                    v2 (모델 메트릭):
Energy — 시간에 따라 감소         ❌ 삭제 (도구를 방해하는 메커닉)
Mood — 방치하면 하락             ❌ 삭제 (죄책감 유발 ≠ 가치)
Focus — 과다 사용 시 감소         ❌ 삭제 (많이 쓸수록 좋아야 함)
Trust — 승패에 따라 축적          → Accuracy로 대체 (실제 메트릭)
```

### 4.2 새 상태 시스템 (전부 실제 모델 메트릭)

| 상태 | 실제 의미 | 측정 방법 | DOUNI 표현 |
|------|----------|----------|-----------|
| **Accuracy** | 최근 N회 예측 정확도 | 매치 결과 rolling average | 높으면 당당, 낮으면 겸손 |
| **Confidence** | 모델의 확신도 보정 | calibration score (예측 70% → 실제 70%?) | 높으면 확신 있게 말함, 낮으면 "아직 모르겠어" |
| **Data Freshness** | 훈련 데이터 최신성 | 마지막 학습 이후 시간 | 신선하면 활발, 오래되면 "새 데이터 줘" |
| **Specialization** | 특정 패턴/자산 전문성 깊이 | 패턴별 sample count + 승률 | 전문 분야에서 흥분, 모르는 분야에서 솔직히 "이건 약해" |

### 4.3 에이전트-도구 정렬 원칙

```
원칙: 모든 에이전트 메커닉은 도구 가치에 정렬되어야 한다.
     "돌보는 행위 = 도구를 쓰는 행위 = 모델을 훈련하는 행위"
     세 가지가 같은 것이어야 한다.

❌ 하지 않는 것:
  • 인위적 성능 감소 (펫 배고픔)
  • 도구 기능 차단 (밥 줘야 분석 가능)
  • 가짜 상태 (실제 모델과 무관한 숫자)

✅ 하는 것:
  • 더 쓸수록 더 좋아짐 (사용 = 보상)
  • 안 쓰면 시장에 뒤처짐 (진짜. 가짜 벌이 아님.)
  • 백그라운드에서 24/7 스캐닝 (유저 없어도 시장을 본다)
  • 놓친 시그널/기회 알려줌 (FOMO → 자연 복귀)
  • 약한 부분 투명 공개 ("하락장 데이터 부족해")
```

### 4.4 방치 = 시장에 뒤처지는 것 (진짜)

```
유저가 3일 안 들어오면:
  시장은 계속 움직였다.
  에이전트는 24/7 스캐닝하고 있었다.
  하지만 유저의 판단 데이터가 없어서 학습은 멈춤.

  복귀 시 DOUNI:
  "3일 동안 시장이 많이 바뀌었어.
   BTC가 레인지 → 하락 추세로 전환됐는데,
   내 학습 데이터는 아직 레인지 기반이야.

   + 네가 없는 동안 내가 감지한 시그널 3개:
   [1] BTC 숏스퀴즈 셋업 (2일 전) → 실제 +4.2% 반등 ✅
   [2] ETH 데스크로스 (어제) → 진행 중 ⏳
   [3] SOL OI 급증 (오늘) → 아직 미확정 🔍

   최신 데이터로 모델 업데이트할까?"
   [업데이트] [시그널 자세히 보기] [나중에]

이건 가짜 Energy 감소가 아님.
실제로:
  • Data Freshness가 떨어져 있음 (모델이 현재 레짐을 모름)
  • 놓친 기회가 있음 (FOMO)
  • 하지만 기존 성능은 유지됨 (벌 없음)
  • 업데이트하면 바로 최신화됨 (회복 쉬움)
```

## 5. 성장 = 자연스러운 모델 발전

### 5.1 Progressive Disclosure (잠금이 아닌 표면 관리)

```
원칙: 기능을 막지 않는다. 노출을 관리한다.
     초보는 10개만 보이고, 나머지는 "더 보기" 뒤에.
     고수는 "더 보기" 누르면 바로 전부 접근.
     에이전트가 사용 패턴을 보고 새 도구를 추천.

인디케이터:
  기본 표시: 10개 (RSI, MACD, BB, MA, Vol, OI, Funding, ATR, Supertrend, F&G)
  "더 보기": 나머지 80+ (잠긴 게 아님. 접힌 것.)

```

캐릭터를 빼면 대시보드 숫자. 캐릭터를 넣으면 대화.

### 9.2 애니메이션 (v1 유지, 의미 변경)

| 상태 | 트리거 | DOUNI 표현 | 실제 의미 |
|------|--------|-----------|----------|
| Idle | 대기 중 | 깜빡임 + 미세 흔들림 | 시장 모니터링 중 |
| Thinking | 분석 요청 | Side + 차트 응시 | 모델 추론 진행 중 |
| Excited | 강한 시그널 | Front + 점프 + 느낌표 | 높은 confidence 시그널 감지 |
| Confident | 맞췄을 때 | 당당한 포즈 | Accuracy ↑ 반영 |
| Humble | 틀렸을 때 | 고개 숙임 + 분석 | 에러 분석 + 학습 중 |
| Alert | 급변동 | 빠른 깜빡임 + 경고 | 실시간 이벤트 감지 |
| Growing | 레벨업 | 진화 애니메이션 | 모델 아키텍처 업그레이드 |

**v1 대비 변경:**
- "Sad/Back" → "Humble" (틀리면 돌아앉는 게 아니라 왜 틀렸는지 분석)
- "Sleep" → 삭제 (방치 페널티 없음)
- "Happy" → "Confident" (감정이 아니라 상태)

### 9.3 성격 분화 (성장 경로 = 모델 특성)

```
성장 경로를 "성격"으로 표현하지만 실제는 모델 설정:

Aggressive (공격형):
  DOUNI 성격: "빠르게 간다! 기회를 놓치지 마!"
  모델 실체: 높은 recall, 낮은 precision. 시그널 많이 잡지만 오탐도 있음.
  적합: 스캘핑, 단타

Balanced (균형형):
  DOUNI 성격: "확실할 때만 가자."
  모델 실체: precision과 recall 균형. 기본 설정.
  적합: 스윙 트레이딩

Conservative (보수형):
  DOUNI 성격: "확실하지 않으면 안 가."
  모델 실체: 높은 precision, 낮은 recall. 시그널 적지만 정확.
  적합: 포지션 트레이딩, 리스크 관리 중시

→ Create에서 선택하지만 언제든 변경 가능
→ 게임 잠금이 아니라 설정
```

## 11. 경쟁 우위와 해자 (정직한 분석)

### 11.1 진짜 해자

```
1. 개인 훈련 데이터 (강함)
   유저 3개월 판단 데이터 = 그 유저에게만 의미 있는 자산
   경쟁사가 가져갈 수 없음
   시간이 갈수록 강화됨

2. 모델 전환 비용 (강함)
   "내 DOUNI를 버리고 0부터 시작?"
   → 불가. 3개월치 훈련을 다시 할 수 없음.

3. 트레이딩 특화 ML 파이프라인 (중간)
   90+ 인디케이터 + multi-stage reasoning + per-user LoRA
   → 범용 AI (ChatGPT, Claude)는 이걸 못 함
   → 하지만 경쟁사가 시간 투자하면 복제 가능

4. DOUNI 캐릭터 + UX (약함)
   캐릭터 자체는 복제 가능
   하지만 "내 모델 + 내 데이터 + 캐릭터 결합"은 감정적 전환 비용
```

### 11.2 진짜 약점

```
1. 범용 AI의 침범
   ChatGPT/Claude가 실시간 데이터 + 차트 접근 얻으면?
   → 대화형 분석은 바로 경쟁
   → 방어: 개인화 + 전문성 (범용은 이걸 못 함)

2. 모델 성능 불확실
   per-user LoRA가 정말 개선되는가?
   → 검증 안 됨. 최대 리스크.
   → 대응: §12에서 다룸

3. 초기 콜드스타트
   모델이 좋아지려면 3개월
   → 3개월을 못 기다리는 유저는 이탈
   → 대응: Base model 자체가 이미 유용해야 함

4. 데이터 API 의존성
   Glassnode, Coinglass 등이 가격 올리면?
   → 대응: 핵심 데이터는 자체 수집 (Binance WebSocket)
           온체인은 프리미엄 기능으로 비용 전가

## 12. 리스크와 대응

### 12.1 치명적 리스크

| # | 리스크 | 확률 | 영향 | 대응 |
|---|--------|------|------|------|
| 1 | per-user LoRA가 실제로 개선 안 됨 | 30% | 치명적 | **런치 전 3개월 내부 테스트 필수.** 5명이 실제 사용하고 정확도 추이 측정. 개선 안 되면 피벗: 개인화 없이 범용 모델 + 대화 UI로 시작. |
| 2 | Base model 자체가 충분히 좋지 않음 | 20% | 치명적 | Arena 결과로 지속 측정. Base가 random(50%)보다 유의미하게 높아야(>55%) 출시. |
| 3 | "대화형 분석"이 트레이더에게 어필 안 됨 | 25% | 높음 | 프로토타입 2주 → 트레이더 5명 테스트. "매일 쓰겠냐?" 4/5 이상이어야 진행. |

### 12.2 구조적 리스크

| # | 리스크 | 대응 |
|---|--------|------|
| 4 | GPU 비용 스케일링 | Free 유저는 Base model만 (GPU 비용 최소). 유료 유저의 LoRA는 배치 업데이트 (주 1회). 실시간 추론은 모델 경량화 (quantization). |
| 5 | 외부 API 의존 (Glassnode 등) | 핵심 데이터(Binance)는 자체 수집. 온체인/파생상품은 프리미엄 기능으로 비용 전가. API 중단 시 대체재 즉시 전환 가능하도록 abstraction layer. |
| 6 | 규제 (투자 자문 해당?) | 에이전트는 "분석 보조 도구"이며 "투자 권유"가 아님을 명시. 실거래 연동 안 함 (페이퍼만). 면책 조항. 법률 검토 후 실거래 연동 결정. |
| 7 | per-user LoRA 저장 비용 | LoRA rank 8 ≈ 25MB, rank 32 ≈ 100MB. 1만 유저 × 100MB = 1TB. S3 비용 ~$23/월. 문제없음. |

### 12.3 검증 순서 (프로토타입 → 런치)

```
Week 1-2: 프로토타입
  Terminal 대화형 UI + DOUNI 반응 + 기본 차트
  → 검증: "대화로 차트 분석하는 게 편한가?"

Week 3-4: 트레이더 테스트
  실제 트레이더 5명 × 30분
  → 검증: "매일 쓰겠냐?" "돈 내겠냐?"

Month 2-3: 내부 LoRA 테스트
  팀 5명이 실제 사용 + per-user LoRA 적용
  → 검증: "정확도가 실제로 올라가는가?"

Month 3-4: 클로즈드 베타

  → 검증: D7 retention > 30%? 유료 전환 > 5%?
